# 第四阶段实施计划：交易终端与高级特性 (Phase 4: Trading & Advanced Features)

## 1. 阶段目标
本阶段完成系统的闭环，实现 **模拟交易 (Paper Trading)** 功能，并完善用户权限与系统配置。
至此，用户不仅能"看"（行情）、"想"（分析），还能"做"（交易）。

## 2. 模块化设计 (Module Specification)

### 2.1 新增模块：`modules/paper_trading`
*   **职责**: 模拟真实的交易所撮合引擎和账户资金管理。
*   **关键特性**: 多币种 (CNY/HKD/USD) 账本，T+1/T+0 规则区分。

### 2.2 增强模块：`modules/user_settings` (Core User 的扩展)
*   **职责**: 提供用户个性化配置和 API Key 管理。
*   **安全性**: 敏感字段加密存储。

## 3. 详细执行步骤 (Task Breakdown)

### 步骤 1: 模拟交易引擎 (Backend - Trading Engine)
*   [ ] **多币种账户体系**:
    *   设计 `PaperAccount` 模型，包含 `cash: { CNY, USD, HKD }`。
    *   实现资金冻结与解冻逻辑（下单时冻结资金）。
*   [ ] **订单撮合系统**:
    *   实现 `OrderService.place_order`。
    *   支持 **市价单** (Market Order) 和 **限价单** (Limit Order)。
    *   **规则引擎**: 针对 A 股实现 T+1 卖出限制，针对港美股实现 T+0。
*   [ ] **清算与结算**:
    *   开发每日收盘后的清算任务，更新持仓成本和总资产。

### 步骤 2: 交易终端界面 (Frontend - Trading Terminal)
*   [ ] **交易面板**: 
    *   仿券商风格的下单界面（买卖方向、价格、数量）。
    *   实时计算"最大可买/可卖数量"。
*   [ ] **持仓管理**: 展示当前持仓、浮动盈亏（需实时刷新价格）。
*   [ ] **订单历史**: 展示当日委托和历史成交记录。

### 步骤 3: 用户与系统设置 (User Settings)
*   [ ] **API Key 管理**: 允许用户绑定自己的 Tushare Token / OpenAI Key。
*   [ ] **偏好设置**: 主题切换、默认市场设置。
*   [ ] **管理员面板**: (如果需要) 简单的用户管理和系统日志查看。

### 步骤 4: 系统集成与全链路测试 (Integration)
*   [ ] **全链路验收**: 
    *   用户注册 -> 充值(模拟) -> 选股 -> AI分析 -> 下单买入 -> 查看持仓 -> 卖出。
*   [ ] **性能优化**: 对行情刷新、资产计算等高频接口进行压测和缓存优化。
*   [ ] **部署准备**: 编写最终的 `Dockerfile` 和部署文档。

## 4. 验收标准
1.  在 A 股市场当日买入的股票，当日无法卖出（T+1 验证）。
2.  下单金额超过可用资金时，系统提示"资金不足"。
3.  交易产生的盈亏能正确反映在仪表盘的总资产中。
