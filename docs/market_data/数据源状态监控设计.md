## 一、功能概述

### 1.1 核心目标

提供数据源健康状态的可观测性，让所有用户（包括管理员和普通用户）能够实时了解各市场、各数据类型的接口工作状态。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **最小粒度降级** | 按数据类型独立判断，日线失败不影响财务数据 |
| **被动状态更新** | 不主动定时检查，在实际调用接口时更新状态 |
| **分级错误展示** | 普通用户看简要，管理员看详细堆栈 |
| **手动重试恢复** | 标记不可用后支持手动重试，成功后自动恢复 |

### 1.3 展示位置

| 位置 | 内容 | 访问权限 |
|------|------|---------|
| **仪表板概览** | 整体健康状态（简化版） | 所有用户 |
| **系统监控页面** | 详细状态（按市场Tab切换） | 所有用户 |

## 五、测试方案

### 5.1 单元测试

| 测试项 | 测试场景 | 预期结果 |
|--------|---------|---------|
| 接口成功 | Mock 正常响应 | status=healthy, failure_count=0 |
| 接口失败（第1次） | Mock 错误响应 | failure_count=1, 状态保持 |
| 接口失败（第3次） | 连续3次Mock错误 | status=unavailable, 触发降级 |
| 降级恢复 | Mock备用源成功 | fallback_info记录, 状态=degraded |
| 手动重试成功 | Mock重试成功 | status=healthy, 清除失败计数 |
| 手动重试失败 | Mock重试失败 | failure_count累加, 保持unavailable |

### 5.2 集成测试

#### 场景1：定时任务降级流程

```
1. 设置 Tushare 返回错误
2. 触发定时同步任务
3. 验证：重试3次
4. 验证：标记为 unavailable
5. 验证：自动降级到 AkShare
6. 验证：状态页面显示降级信息
```

#### 场景2：工具调用立即降级

```
1. 设置 Tushare 返回错误
2. TradingAgents 工具调用数据接口
3. 验证：不重试，立即降级
4. 验证：返回 AkShare 数据
```

#### 场景3：手动重试恢复

```
1. 设置数据源为 unavailable
2. 用户点击"重试"按钮
3. 修复 Tushare，返回正常数据
4. 验证：状态恢复为 healthy
5. 验证：如果之前使用备用源，切换回主源
```

#### 场景4：多接口数据类型

```
1. 设置 Tushare pro_bar 失败
2. 设置 Tushare daily 正常
3. 调用日线数据接口
4. 验证：尝试使用 daily 补偿
5. 验证：状态显示 degraded（部分数据）
```
