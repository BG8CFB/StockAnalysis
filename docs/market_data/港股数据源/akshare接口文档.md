## 数据源简介

**AKShare** 是基于 Python 的开源财经数据接口库，完全免费，无需 API Key。

### 核心特性

- 免费开源：完全免费，无需 API Key
- 数据覆盖广：支持港股市场全覆盖
- 接口丰富：提供港股专用接口
- 简单易用：统一接口设计，返回 pandas DataFrame 格式

## 港股接口

### 1. 实时行情数据

#### 1.1 港股实时行情

**接口名称**：`stock_hk_spot()`

**功能描述**：获取港股所有上市公司的实时行情数据

**数据源**：东方财富网

**返回字段**：

| 字段名 | 说明 | 字段名 | 说明 |
|--------|------|--------|------|
| 代码 | 股票代码 | 名称 | 股票名称 |
| 最新价 | 最新成交价 | 涨跌幅 | 涨跌幅百分比 |
| 涨跌额 | 涨跌金额 | 成交量 | 成交量（手） |
| 成交额 | 成交额（元） | 振幅 | 振幅百分比 |
| 最高 | 最高价 | 最低 | 最低价 |
| 今开 | 今开价 | 昨收 | 昨收价 |
| 量比 | 量比 | 换手率 | 换手率 |

**示例代码**：

```python
import akshare as ak

# 获取所有港股实时行情
df = ak.stock_hk_spot()

# 筛选特定股票（如腾讯）
stock_info = df[df['代码'] == '00700']
print(stock_info)
```

### 2. 历史行情数据

#### 2.1 港股历史日线数据

**接口名称**：`stock_hk_daily()`

> **测试状态**：⚠️ 测试失败 (2026-01-03测试)
> **问题**：接口不接受 `start_date`/`end_date` 参数，与文档描述不一致
> **错误信息**：`stock_hk_daily() got an unexpected keyword argument 'start_date'`
> **建议**：使用 Yahoo Finance 或 TuShare 获取港股历史数据

**功能描述**：获取港股个股历史日线行情数据

**数据源**：东方财富网

**⚠️ 注意事项**：
- 实际接口参数与文档描述不一致
- `start_date` 和 `end_date` 参数不被支持
- 建议使用其他数据源获取港股历史数据

**请求参数**（实际可用）：

| 参数名 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| symbol | str | 股票代码（如 "00700"） | - |

**文档描述参数**（实际不可用）：

| 参数名 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| start_date | str | 开始日期 "19900101" | "19900101" |
| end_date | str | 结束日期 "21000101" | "21000101" |
| adjust | str | 复权类型："" 不复权, "qfq" 前复权 | "" |

**返回字段**：

| 字段名 | 说明 |
|--------|------|
| 日期 | 交易日期 |
| 开盘 | 开盘价 |
| 收盘 | 收盘价 |
| 最高 | 最高价 |
| 最低 | 最低价 |
| 成交量 | 成交量（手） |
| 成交额 | 成交额（元） |

**示例代码**：

```python
import akshare as ak

# 获取腾讯控股历史日线数据（前复权）
df = ak.stock_hk_daily(
    symbol="00700",
    start_date="20200101",
    end_date="20241231",
    adjust="qfq"
)
print(df.head())
```

## 限流与注意事项

### 1. 请求频率限制

AKShare 本身不强制限流，但依赖的第三方数据源有反爬机制。

**建议策略**：
- 单个接口请求间隔：0.5-1 秒
- 批量请求时添加随机延时
- 避免在短时间内重复请求相同数据

### 2. 并发控制

**推荐并发数**：
- 同一域名：不超过 3 个并发
- 不同数据源：可适当提高

### 3. 错误重试机制

**建议配置**：
- 最大重试次数：3 次
- 重试间隔：指数退避（1s, 2s, 4s）

## 常见问题与解决方案

### 1. ReadTimeout 错误

**解决方案**：
1. 重新运行接口函数
2. 降低访问频率
3. 增加超时时间

### 2. HTTP 403 错误

**原因**：请求过于频繁，触发反爬机制

**解决方案**：
1. 降低请求频率
2. 添加请求延时
