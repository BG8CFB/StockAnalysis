**职责**：数据源配置的创建、读取、更新、删除（管理员专用）

### 3. sources/ - 数据源适配器

| 文件 | 说明 |

| `metadata/` | 数据源元数据目录 |
| `metadata/sources_manifest.yaml` | 数据源清单（支持的数据源列表） |

| 文件 | 说明 |
|------|------|
| `tushare_adapter.py` | TuShare 数据源适配器 |
| `akshare_adapter.py` | AkShare 数据源适配器 |

#### 3.3 us_stock/ - 美股数据源

| 文件 | 说明 |
|------|------|
| `yahoo_adapter.py` | Yahoo Finance 数据源适配器 |
| `alphavantage_adapter.py` | Alpha Vantage 数据源适配器 |

#### 3.4 hk_stock/ - 港股数据源

| 文件 | 说明 |
|------|------|

- 适配器负责：API 调用、字段映射、数据标准化

## 数据源适配器设计原则

### 为什么按市场分目录

- 不同市场的数据特点完全不同（A股/美股/港股）
- 每个市场可能有 3-4 个数据源
- 每个适配器文件负责一个数据源
- 便于管理和维护，新增数据源时只需添加对应市场的适配器文件

### 每个适配器的职责

1. 封装对应数据源的 API 调用
2. 实现字段映射（原始字段 → 标准字段）
3. 实现数据类型转换
4. 实现单位标准化（手→股，千元→元）
5. 实现股票代码标准化
6. 实现数据验证

### 适配器接口规范

所有适配器必须实现的方法：

```
DataSourceAdapter（抽象基类）
├─ get_daily_quote(symbol, date)        # 获取单日行情
├─ get_klines(symbol, start, end)       # 获取历史K线
├─ get_financials(symbol, period)       # 获取财务报表
├─ get_company_info(symbol)             # 获取公司信息
├─ get_news(symbol, start, end)         # 获取新闻
└─ normalize_data(raw_data, data_type)  # 数据标准化
```

## 扩展指南

### 新增数据源

1. 在对应市场的 `sources/{market}/` 目录下创建 `{source}_adapter.py`
2. 继承 `DataSourceAdapter` 基类
3. 实现所有必需方法
4. 在 `registry.py` 中注册数据源
5. 在 `metadata/sources_manifest.yaml` 中添加数据源信息

### 新增数据类型

1. 在 `models/` 中创建对应的模型文件
2. 在 `repositories/` 中创建对应的 Repository
3. 在适配器中实现获取该数据类型的方法
4. 更新工具模块，添加对应的工具函数

### 新增市场

1. 创建 `sources/{market}/` 目录
2. 实现对应市场的数据源适配器
3. 在 `models/` 中添加该市场的数据模型
4. 在 `tools/` 中创建该市场的工具集
5. 更新前端路由和页面
