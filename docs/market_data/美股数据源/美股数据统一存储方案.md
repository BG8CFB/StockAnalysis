## 一、数据分类与更新频率

| 数据大类 | 数据内容 | 优先级 | 更新频率 |
|---------|---------|-------|---------|
| **基础信息** | 股票列表、公司概况 | 高 | 每周 |
| **行情数据** | 日K线、分钟K线、实时行情 | 高 | 每日/实时 |
| **财务报表** | 利润表、资产负债表、现金流量表 | 中 | 每季度 |
| **财务指标** | PE、PB、EPS等 | 中 | 每日 |
| **公司行为** | 拆股、分红 | 低 | 不定期 |
| **市场数据** | 指数、行业板块 | 低 | 每日 |

## 三、存储策略

### 3.1 集合设计

| 集合名 | 说明 | 索引设计 | 备注 |
|-------|------|---------|------|
| `stock_info` | 股票基础信息 | `(market, symbol)` 唯一 | |
| `stock_quotes` | 日线/分钟行情 | `(market, symbol, trade_date)` 唯一 | 包含多市场数据 |
| `stock_financials` | 财务报表 | `(market, symbol, report_date, report_type)` | |
| `stock_dividends` | 分红拆股 | `(market, symbol, date)` | |

### 3.2 数据源优先级

**原则**：
1. **日线行情**：首选 **AkShare**（无限制，需本地过滤日期），备用 Yahoo Finance。
2. **实时行情**：首选 **AkShare** (`stock_us_spot`)。
3. **财务数据**：首选 **Yahoo Finance**（数据更全），备用 AkShare。
4. **分钟数据**：首选 **Yahoo Finance**（仅近30天），AkShare (如有) 为辅。

## 四、数据同步策略（美股特有）

### 4.1 "自选股优先"策略

鉴于美股股票数量巨大（10000+）且数据源限制：

1.  **Watchlist Sync (高频)**：
    *   对象：用户添加到"自选股"的股票。
    *   频率：每日收盘后自动同步。
    *   方式：AkShare 全量获取 + 本地截取最新日期。

2.  **On-Demand Fetch (按需)**：
    *   对象：用户临时查询的非自选股。
    *   触发：用户查看详情或添加到自选时。
    *   方式：实时调用接口获取并缓存。

3.  **Market Overview (概览)**：
    *   对象：指数 (.INX, .IXIC)、热门股。
    *   频率：每日同步。

### 4.2 差异处理逻辑

*   **AkShare 参数限制**：由于 `stock_us_daily` 不支持时间范围，系统需实现 `fetch_and_filter` 逻辑：
    ```python
    # 伪代码
    df = ak.stock_us_daily(symbol="AAPL")
    df = df[df['date'] >= start_date] # 本地过滤
    ```
*   **Yahoo 限流**：系统需维护 `TokenBucket` 限流器，严格控制对 Yahoo 的调用频率（<2000次/小时）。

---

## 五、异常处理

1.  **AkShare 超时**：重试 3 次，若失败则切换 Yahoo。
2.  **Yahoo 403/429**：立即停止 Yahoo 调用 15 分钟，仅使用 AkShare。
3.  **数据缺失**：若某日数据缺失，标记为 `suspect`，尝试从另一数据源补录。
