# 权限系统升级建议 (RBAC)

## 1. 权限模型设计

采用标准的 **RBAC Level 1 (Role-Based Access Control)** 模型，引入“角色”作为用户和权限的中间层。

### 1.1 实体关系
*   **User (用户)**：多对多 -> **Role (角色)**
*   **Role (角色)**：多对多 -> **Permission (权限)**

### 1.2 数据结构

**Role 集合**:
```json
{
  "_id": "role_admin",
  "name": "管理员",
  "description": "系统超级管理员，拥有所有权限",
  "permissions": ["*"],  // 通配符表示所有
  "is_system": true      // 禁止删除
}
```

**Permission 定义 (代码常量)**:
权限字符串采用 `resource:action` 格式，例如：
*   `analysis:create`: 创建分析任务
*   `analysis:view_history`: 查看历史记录
*   `system:config_read`: 读取系统配置
*   `system:config_write`: 修改系统配置
*   `users:manage`: 管理其他用户

### 1.3 预置角色

| 角色标识 | 名称 | 权限集描述 | 适用人群 |
| :--- | :--- | :--- | :--- |
| `admin` | 管理员 | `*` (所有权限) | 运维/系统拥有者 |
| `vip_user` | 高级会员 | `analysis:*`, `paper_trading:*`, `market:us_hk` (港美股) | 付费用户 |
| `user` | 普通用户 | `analysis:create` (受限), `paper_trading:basic` | 注册用户 |
| `guest` | 访客 | `market:view` (延迟行情) | 未登录/演示 |

## 2. 鉴权流程实现

### 2.1 装饰器鉴权 (FastAPI Depends)

实现一个依赖注入函数 `check_permission`：

```python
from fastapi import Depends, HTTPException
from app.routers.auth_db import get_current_user

def require_permission(permission_code: str):
    async def dependency(user: dict = Depends(get_current_user)):
        # 1. 获取用户角色
        role_ids = user.get("role_ids", [])
        
        # 2. 查询角色对应的权限 (通常缓存到 Redis)
        user_permissions = await get_permissions_by_roles(role_ids)
        
        # 3. 校验
        if "*" in user_permissions or permission_code in user_permissions:
            return user
            
        raise HTTPException(status_code=403, detail="权限不足")
    return dependency

# 使用示例
@router.post("/system/config")
async def update_config(
    request: ConfigRequest, 
    user: dict = Depends(require_permission("system:config_write"))
):
    ...
```

### 2.2 数据级权限控制 (Row-Level Security)

对于某些不仅需要“功能权限”还需要“数据范围权限”的场景（例如管理员查看所有用户任务，普通用户只能看自己的）：

```python
async def get_analysis_list(user: dict, page: int):
    query = {}
    
    # 如果不是管理员，强制加上 user_id 过滤
    if not has_role(user, "admin"):
        query["user_id"] = user["id"]
        
    return await db.analysis_tasks.find(query).skip(...).limit(...)
```

## 3. 实施步骤

1.  **定义权限字典**：在代码中创建一个常量文件 `app/constants/permissions.py`，列出所有合法的权限字符串。
2.  **初始化角色**：编写数据库迁移脚本，在 `roles` 集合中插入默认的 `admin` 和 `user` 角色。
3.  **改造 User 模型**：添加 `role_ids` 字段，并将旧的 `is_admin=True` 映射为 `role_ids=["admin"]`。
4.  **应用鉴权**：在 `app/routers/system_config.py` 等敏感接口上添加 `@Depends(require_permission(...))`。
