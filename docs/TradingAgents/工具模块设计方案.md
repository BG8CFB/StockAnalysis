# TradingAgents 工具模块技术文档

本文档描述 TradingAgents 模块中可供 AI 智能体推理时调用的工具接口。

本文档分为三部分：
1. **工具接口说明**：描述 TradingAgents 智能体可调用的工具，以及其输入输出格式和实现逻辑。
2. **底层数据接口 (TU API) 说明**：描述工具内部调用的底层数据源接口（如 Tushare、Binance 等）。
3. **工具模块与数据模块的集成**：描述工具如何调用数据模块获取数据。

---

## 工具模块与数据模块的集成

### 集成概述

工具模块通过调用数据模块（`backend/modules/market_data/`）获取股票数据，而不是直接调用外部数据源接口。

**核心原则**：
- 工具模块只负责数据格式转换和业务逻辑处理
- 数据获取、缓存、限流等逻辑由数据模块统一处理
- 工具模块通过双通道机制获取数据（实时信息源 vs 项目信息源）

### 数据源模式

工具调用时需指定 `data_source_mode` 参数：

| 模式 | 说明 | 数据来源 | 使用场景 |
|------|------|---------|---------|
| `realtime` | 实时信息源 | 用户配置的付费接口（双通道） | 需要最新实时数据 |
| `project` | 项目信息源 | 系统数据库（免费数据源） | 常规分析，接受数据略有延迟 |

### 双通道数据流

#### 通道1（直接返回）

**特点**：数据直接返回给工具模块，不存储，不限流

**流程**：
```
用户配置的付费接口
    ↓
获取数据
    ↓
直接返回给工具模块
    ↓
提供给 AI 智能体使用
```

#### 通道2（存入数据库）

**特点**：数据存入系统数据库，按股票+数据类型+1分钟限流

**流程**：
```
用户配置的付费接口
    ↓
获取数据
    ↓
检查限流记录（股票+数据类型）
    ├─ 未过期 → 丢弃，不写入
    └─ 已过期 → 写入数据库 + 记录限流时间
```

### 工具调用流程

#### 场景1：用户选择"项目信息源"

```
AI 智能体调用工具（data_source_mode="project"）
    ↓
工具调用数据模块的查询接口
    ↓
数据模块从数据库读取数据
    ├─ 存在且未过期 → 直接返回
    └─ 不存在或已过期 → 触发接口更新后返回
    ↓
工具返回数据给 AI 智能体
```

#### 场景2：用户选择"实时信息源"（已配置付费接口）

```
AI 智能体调用工具（data_source_mode="realtime"）
    ↓
工具调用数据模块的实时接口
    ↓
数据模块检查用户是否配置了该市场的实时数据源
    ├─ 未配置 → 使用"项目信息源"（从数据库获取）
    │
    └─ 已配置 → 调用用户配置的付费接口
        ├─ 调用成功
        │   ├─ 通道1：直接返回（不限流）
        │   └─ 通道2：尝试写入数据库（限流）
        │
        └─ 调用失败 → 降级到"项目信息源"（从数据库获取）
    ↓
工具返回数据给 AI 智能体
```

### 数据获取的优先级

**数据来源优先级**（从高到低）：

1. **用户实时数据源（通道1）**：用户配置的付费接口，直接返回
2. **系统数据库（项目信息源）**：系统公共数据源的数据
3. **降级策略**：用户接口失败时自动降级到数据库

### 并发控制

**限流粒度**：按股票代码 + 数据类型 + 1分钟时间窗口

**场景**：100个用户同时分析000001.SZ

```
时间线：
├─ T0秒：100个用户同时发起分析（都使用实时信息源）
├─ T0.1秒：100个用户都调用付费接口（通道1，不限流）
├─ T0.5秒：100个用户都获取到数据
│
├─ 通道1（直接返回，不限流）：
│   └─ 100个用户的数据 → 工具模块 → AI分析（并行执行）
│
└─ 通道2（存入数据库，限流）：
    ├─ 第1个用户尝试写入000001.SZ的quote数据
    │   ├─ 检查限流记录：不存在
    │   ├─ 允许写入 → 写入成功
    │   └─ 记录限流：000001.SZ + quote（过期时间1分钟）
    │
    └─ 其余99个用户尝试写入
        ├─ 检查限流记录：存在且未过期
        ├─ 拒绝写入 → 数据丢弃
        └─ 99个用户的通道2数据不写入数据库
```

### 数据一致性

#### 盘中数据 vs 收盘数据

**盘中数据**：
- 交易时间内获取的数据（如14:30）
- 数据字段 `is_complete: false`（未收盘）
- 可能是未完成的当日行情

**收盘数据**：
- 收盘后获取的完整数据（15:30后）
- 数据字段 `is_complete: true`（已收盘）
- 完整的日K线数据

**查询优先级**：
- 用户使用"项目信息源"时优先返回 `is_complete: true` 的数据
- 用户使用"实时信息源"时返回接口实时返回的数据（可能是盘中数据）

### API 调用示例

**示例1：获取股票行情（项目信息源）**

```
工具调用：
{
  "tool": "get_stock_quote",
  "params": {
    "symbol": "000001.SZ",
    "market_type": "stock",
    "start_date": "20230101",
    "end_date": "20230131",
    "data_source_mode": "project"  // 项目信息源
  }
}

数据流：
工具 → 数据模块查询接口 → 数据库 → 返回数据
```

**示例2：获取股票行情（实时信息源）**

```
工具调用：
{
  "tool": "get_stock_quote",
  "params": {
    "symbol": "000001.SZ",
    "market_type": "stock",
    "start_date": "20230101",
    "end_date": "20230131",
    "data_source_mode": "realtime",  // 实时信息源
    "user_id": "user_123"
  }
}

数据流：
工具 → 数据模块实时接口
    ├─ 检查用户配置 → 调用付费接口
    ├─ 通道1：直接返回（给AI使用）
    └─ 通道2：限流后写入数据库（供其他用户使用）
```

### 降级逻辑

**用户配置了实时数据源但调用失败时**：

```
工具调用实时接口（data_source_mode="realtime"）
    ↓
数据模块尝试调用用户配置的接口
    ↓
调用失败（网络错误、API Key失效等）
    ↓
自动降级到"项目信息源"
    ↓
从数据库获取数据
    ↓
返回给工具模块
```

## 第一部分：工具接口说明 (Tools)

本部分详细说明 TradingAgents 智能体在推理过程中可调用的工具。

### 1. 市场行情类工具

#### 1.1 `stock_data`
*   **功能**: 获取股票、期货、外汇等多种市场的日线行情数据，并支持计算技术指标。
*   **输入格式 (Input)**:
    ```json
    {
      "code": "000001.SZ",
      "market_type": "stock",
      "start_date": "20230101",
      "end_date": "20230131",
      "indicators": ["MACD", "KDJ", "MA"]
    }
    ```
*   **输出格式 (Output)**:
    ```json
    [
      {
        "trade_date": "20230131",
        "open": 14.2,
        "close": 14.5,
        "high": 14.6,
        "low": 14.1,
        "vol": 100000,
        "amount": 1450000,
        "ma5": 14.3,
        "macd": 0.12,
        "k": 45.2,
        "d": 40.1,
        "j": 55.4
      }
      // ... 更多日期数据
    ]
    ```
*   **TU 接口调用说明**:
    1.  **获取基础行情**:
        *   若 `market_type` 为 `stock` (A股)，调用 Tushare 接口 **`daily`**。
        *   若 `market_type` 为 `crypto` (加密货币)，调用 Binance 接口 **`/api/v3/klines`**。
        *   其他市场类型分别调用对应的 Tushare 接口（见下方列表）。
    2.  **获取复权因子 (仅A股)**:
        *   调用 Tushare 接口 **`adj_factor`**，用于计算前复权价格。
    3.  **内部计算**:
        *   利用获取的 OHLCV 数据，在本地计算 MACD, KDJ, RSI, BOLL, MA 等指标。

#### 1.2 `stock_data_minutes`
*   **功能**: 获取分钟级 K 线数据。
*   **输入格式 (Input)**:
    ```json
    {
      "code": "000001.SZ",
      "market_type": "stock",
      "start_datetime": "2023-01-01 09:30:00",
      "end_datetime": "2023-01-01 15:00:00",
      "freq": "1min"
    }
    ```
*   **输出格式 (Output)**:
    ```json
    [
      {
        "trade_time": "2023-01-01 09:30:00",
        "open": 14.2,
        "close": 14.3,
        "high": 14.3,
        "low": 14.1,
        "vol": 5000,
        "amount": 71000
      }
    ]
    ```
*   **TU 接口调用说明**:
    *   A股：调用 Tushare 接口 **`stk_mins`**。
    *   加密货币：调用 Binance 接口 **`/api/v3/klines`**。

#### 1.3 `index_data`
*   **功能**: 获取指数日线行情。
*   **输入格式 (Input)**:
    ```json
    {
      "code": "000300.SH",
      "start_date": "20230101",
      "end_date": "20230131"
    }
    ```
*   **输出格式 (Output)**:
    ```json
    [
      {
        "trade_date": "20230131",
        "open": 3900.5,
        "close": 3950.2,
        "change": 49.7,
        "pct_chg": 1.27
      }
    ]
    ```
*   **TU 接口调用说明**:
    *   调用 Tushare 接口 **`index_daily`**。

#### 1.4 `csi_index_constituents`
*   **功能**: 深度分析中证指数成分股，包含行情、权重、估值和财务数据。
*   **输入格式 (Input)**:
    ```json
    {
      "index_code": "000300.SH",
      "start_date": "20230101",
      "end_date": "20230131"
    }
    ```
*   **输出格式 (Output)**:
    ```json
    {
      "index_summary": { "open": 3900, "close": 3950, "pct_chg": 1.28 },
      "constituents": [
        {
          "code": "600519.SH",
          "name": "贵州茅台",
          "weight": 5.2,
          "price_summary": { "open": 1700, "close": 1750, "pct_chg": 2.9 },
          "valuation": { "pe_ttm": 30.5, "pb": 8.2, "dv_ratio": 1.2 },
          "financial": { "roe": 25.4, "netprofit_margin": 48.2 }
        }
      ]
    }
    ```
*   **TU 接口调用说明**:
    这是一个组合型工具，依次调用了以下接口：
    1.  调用 **`index_daily`**: 获取指数本身的行情。
    2.  调用 **`index_weight`**: 获取指数的成分股及其权重。
    3.  调用 **`daily`**: 批量获取所有成分股的区间行情。
    4.  调用 **`daily_basic`**: 获取成分股的估值数据 (PE, PB 等)。
    5.  调用 **`fina_indicator`**: 获取成分股的财务指标 (ROE, 净利率等)。
    6.  调用 **`dividend`**: 获取成分股的分红数据，用于计算分红率。

### 2. 宏观与基本面工具

#### 2.1 `macro_econ`
*   **功能**: 获取各类宏观经济指标。
*   **输入格式 (Input)**:
    ```json
    {
      "indicator": "gdp", // 或 cpi, ppi, shibor 等
      "start_date": "20230101",
      "end_date": "20231231"
    }
    ```
*   **输出格式 (Output)**:
    ```json
    [
      {
        "quarter": "2023Q1",
        "gdp": 284997,
        "gdp_pc": 20188,
        "pct_chg": 4.5
      }
    ]
    ```
*   **TU 接口调用说明**:
    根据 `indicator` 参数的不同，映射调用不同的 TU 接口：
    *   `shibor`: 调用 **`shibor`** 和 **`shibor_quote`**。
    *   `lpr`: 调用 **`lpr_data`**。
    *   `gdp`: 调用 **`cn_gdp`**。
    *   `cpi`: 调用 **`cn_cpi`**。
    *   `ppi`: 调用 **`cn_ppi`**。
    *   `m0`/`m1`/`m2`: 调用 **`cn_m`**。
    *   `pmi`: 调用 **`cn_pmi`**。
    *   `sf`: 调用 **`cn_sf`**。

#### 2.2 `company_performance` (系列)
包含 `company_performance` (A股), `company_performance_hk` (港股), `company_performance_us` (美股)。
*   **功能**: 获取上市公司的财务报表、业绩预告、分红等基本面数据。
*   **输入格式 (Input)**:
    ```json
    {
      "ts_code": "000001.SZ",
      "data_type": "income", // 或 balance, cashflow, forecast 等
      "start_date": "20230101",
      "end_date": "20231231"
    }
    ```
*   **输出格式 (Output)**:
    ```json
    [
      {
        "end_date": "20231231",
        "revenue": 1000000,
        "net_profit": 200000,
        "eps": 1.2
      }
    ]
    ```
*   **TU 接口调用说明**:
    根据 `data_type` 映射调用 Tushare 对应接口：
    *   A股: 调用 **`income`**, **`balancesheet`**, **`cashflow`**, **`forecast`**, **`express`**, **`fina_indicator`**, **`dividend`** 等。
    *   港股: 调用 **`hk_income`**, **`hk_balancesheet`**, **`hk_cashflow`**。
    *   美股: 调用 **`us_income`**, **`us_balancesheet`**, **`us_cashflow`**, **`us_fina_indicator`**。

### 3. 基金与专项数据工具

#### 3.1 `fund_data`
*   **功能**: 获取基金净值、持仓、经理等数据。
*   **TU 接口调用说明**:
    *   `nav`: 调用 **`fund_nav`**。
    *   `portfolio`: 调用 **`fund_portfolio`**。
    *   `manager`: 调用 **`fund_manager`**。

#### 3.2 `fund_manager_by_name`
*   **功能**: 搜索基金经理并聚合其信息。
*   **TU 接口调用说明**:
    1.  调用 **`fund_manager`** 获取包含该姓名的所有记录。
    2.  **内部处理**: 对返回记录进行智能分组（通过姓名、性别、简历相似度比对），将同一人的多条任职记录合并为一个对象输出。

#### 3.3 `finance_news`
*   **功能**: 搜索财经新闻。
*   **TU 接口调用说明**:
    *   调用 **Baidu Search API** (非 Tushare)。

#### 3.4 `hot_news_7x24`
*   **功能**: 获取 7x24 小时快讯并去重。
*   **TU 接口调用说明**:
    1.  调用 Tushare 接口 **`news`** 获取原始新闻流。
    2.  **内部处理**: 使用 Jaccard 相似度算法对新闻内容进行比对，去除内容高度相似（相似度>0.8）的重复新闻。

## 第二部分：底层数据接口说明 (TU APIs)

本部分详细列出工具内部调用的 Tushare Pro API 接口规范。

> **说明**：TU = Tushare，本项目中所有底层数据接口统一称为 TU 接口，不再使用 "PU" 称谓。

### 1. 行情类 TU 接口

#### 1.1 `daily`
*   **功能**: A股日线行情
*   **发送数据 (Request)**:
    ```json
    {
      "ts_code": "股票代码 (如 000001.SZ)",
      "start_date": "开始日期 (YYYYMMDD)",
      "end_date": "结束日期 (YYYYMMDD)"
    }
    ```
*   **返回内容 (Response)**:
    *   `trade_date`: 交易日期
    *   `open`: 开盘价
    *   `high`: 最高价
    *   `low`: 最低价
    *   `close`: 收盘价
    *   `vol`: 成交量

#### 1.2 `adj_factor`
*   **功能**: A股复权因子
*   **发送数据**: 同 `daily`
*   **返回内容**:
    *   `adj_factor`: 复权因子 (用于计算真实价格)

#### 1.3 `stk_mins`
*   **功能**: A股分钟行情
*   **发送数据**:
    ```json
    {
      "ts_code": "股票代码",
      "start_date": "开始时间 (YYYY-MM-DD HH:MM:SS)",
      "end_date": "结束时间",
      "freq": "1min/5min..."
    }
    ```
*   **返回内容**: `trade_time`, `open`, `close`, `high`, `low`, `vol` 等。

#### 1.4 `index_daily`
*   **功能**: 指数日线行情
*   **发送数据**: `ts_code` (指数代码), `start_date`, `end_date`
*   **返回内容**: `trade_date`, `open`, `close`, `pct_chg` (涨跌幅) 等。

#### 1.5 `index_weight`
*   **功能**: 指数成分股权重
*   **发送数据**: `index_code` (指数代码), `start_date`, `end_date`
*   **返回内容**: `con_code` (成分股代码), `weight` (权重)。

### 2. 财务与基本面 TU 接口

#### 2.1 `daily_basic`
*   **功能**: A股每日指标 (估值数据)
*   **发送数据**: `ts_code`, `trade_date`
*   **返回内容**: `pe_ttm` (滚动市盈率), `pb` (市净率), `dv_ratio` (股息率)。

#### 2.2 `fina_indicator`
*   **功能**: A股财务指标
*   **发送数据**: `ts_code`, `start_date`, `end_date`
*   **返回内容**: `roe`, `grossprofit_margin` (毛利率), `netprofit_margin` (净利率), `debt_to_assets` (资产负债率)。

#### 2.3 `income` / `balancesheet` / `cashflow`
*   **功能**: 三大财务报表
*   **发送数据**: `ts_code`, `start_date`, `end_date`
*   **返回内容**: 标准财务报表字段 (如 `revenue`, `total_assets`, `n_cashflow_act`)。

#### 2.4 `dividend`
*   **功能**: 分红送转数据
*   **发送数据**: `ts_code`
*   **返回内容**: `cash_div` (派现金额), `ex_div_date` (除权除息日)。

### 3. 宏观经济 TU 接口

#### 3.1 `cn_gdp` / `cn_cpi` / `cn_ppi`
*   **功能**: 中国宏观经济数据
*   **发送数据**: `start_date`, `end_date`
*   **返回内容**:
    *   `cn_gdp`: `quarter` (季度), `gdp` (数值), `pct_chg` (同比)。
    *   `cn_cpi`: `month` (月份), `cpi` (数值), `mom` (环比)。

#### 3.2 `shibor` / `lpr_data`
*   **功能**: 利率数据
*   **发送数据**: `start_date`, `end_date`
*   **返回内容**: `date`, `on` (隔夜), `1y` (1年期) 等利率数值。

### 4. 基金与新闻 TU 接口

#### 4.1 `fund_nav`
*   **功能**: 基金净值
*   **发送数据**: `ts_code`, `start_date`, `end_date`
*   **返回内容**: `unit_nav` (单位净值), `accum_nav` (累计净值)。

#### 4.2 `fund_portfolio`
*   **功能**: 基金持仓
*   **发送数据**: `ts_code`
*   **返回内容**: `stock_code` (股票代码), `market_value` (持仓市值), `pct` (占比)。

#### 4.3 `news`
*   **功能**: 新闻快讯
*   **发送数据**: `start_date`, `end_date`, `src` (来源)
*   **返回内容**: `datetime`, `title`, `content`, `channels`。

### 5. 其他 API

#### 5.1 Binance `klines`
*   **功能**: 加密货币 K 线
*   **发送数据**: `symbol` (如 BTCUSDT), `interval` (如 1d), `startTime`, `endTime`
*   **返回内容**: 数组形式 `[Open time, Open, High, Low, Close, Volume, ...]`

---

**文档说明**:
*   **工具 (Tools)** 是 TradingAgents 智能体在推理过程中可调用的函数。
*   **TU API** 是工具内部去 Tushare 或 Binance 获取数据时调用的接口。
*   工具通常会对 TU API 返回的数据进行清洗、计算（如 `stock_data` 计算 MACD）或组合（如 `csi_index_constituents` 组合了5个接口的数据）后，再返回给智能体。
