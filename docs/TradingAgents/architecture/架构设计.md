## 一、架构概述

### 1.1 核心理念

TradingAgents 模块模拟专业投资分析团队的工作模式，通过结构化的多阶段分析流程，对投资标的进行深度研究，最终形成兼具深度、广度和风险考量的投资决策建议。

**设计原则**：
- **简单直接**：使用 Python 原生 async/await 控制流，无需复杂编排层
- **清晰可读**：调度文件一目了然，易于维护和调试
- **高效执行**：利用 asyncio 实现真正的并行处理
- **组件解耦**：每个阶段独立运行，易于测试和扩展

**核心工作模式**：
- **并行研究**：多个不同领域的分析师同时从各自的专业角度进行初步分析
- **交叉辩论**：设立正反方角色，通过多轮辩论深挖潜在机会和风险
- **多层决策**：设置"经理"角色，对下级辩论进行总结和裁决
- **风险前置**：在形成最终决策前，进行独立的多视角风险评估
- **统一输出**：汇总所有过程信息，生成结构化的最终报告

## 三、核心组件

### 3.1 任务管理器 (TaskManager)

**职责**：
- 创建和初始化任务
- 管理任务生命周期（pending → running → completed/failed）
- 处理任务取消和停止
- 任务状态查询和更新
- 批量任务管理

**核心方法**：

| 方法 | 描述 |
|------|------|
| `create_task()` | 创建单个分析任务 |
| `create_batch_task()` | 创建批量分析任务 |
| `get_task_status()` | 获取任务状态 |
| `list_tasks()` | 列出用户任务 |
| `cancel_task()` | 取消任务 |
| `stop_task()` | 停止运行中的任务 |
| `execute_analysis_workflow()` | 执行分析工作流（调用 scheduler） |
| `complete_task()` | 标记任务完成 |
| `fail_task()` | 标记任务失败 |

**批量任务失败处理**：

批量任务中单个子任务失败时，系统自动执行以下处理：

- **自动重试**：失败任务自动重试，最多重试 3 次
- **重试间隔**：第1次立即重试，第2次等待 5 秒，第3次等待 15 秒
- **失败标记**：重试 3 次仍失败后，标记该子任务状态为 `failed`
- **其他继续**：单个子任务失败不影响其他子任务执行
- **部分完成**：批量任务可以部分成功，用户可查看成功和失败的任务列表
- **失败汇总**：批量任务完成后，汇总失败原因和任务列表

### 3.2 并发控制器 (ConcurrencyController)

**职责**：
- 管理模型级别的并发控制
- 实现任务排队机制（FIFO）
- 处理并发槽位的申请和释放
- 支持批量任务并发限制

**三层并发架构**：

```
┌────────────────────────────────────────────────────────────┐
│                    并发控制层级                            │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  ┌──────────────────────────────────────────────────────┐ │
│  │ 第一层：模型级并发 (max_concurrency)                  │ │
│  │ 限制对单个 AI 模型的最大并发请求数                    │ │
│  │ 例如：max_concurrency=40                              │ │
│  └──────────────────────────────────────────────────────┘ │
│                            ↓                               │
│  ┌──────────────────────────────────────────────────────┐ │
│  │ 第二层：任务级并发 (task_concurrency)                 │ │
│  │ 定义单个任务消耗的并发槽位数                           │ │
│  │ 可同时运行的任务数 = max_concurrency / task_concurrency│ │
│  │ 例如：task_concurrency=2 → 可运行 20 个任务           │ │
│  └──────────────────────────────────────────────────────┘ │
│                            ↓                               │
│  ┌──────────────────────────────────────────────────────┐ │
│  │ 第三层：批量任务并发 (batch_concurrency)               │ │
│  │ 限制批量任务中同时执行的子任务数                       │ │
│  │ 例如：batch_concurrency=5 → 批量任务最多运行 5 个子任务│ │
│  └──────────────────────────────────────────────────────┘ │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 3.3 工作流调度器 (WorkflowScheduler)

**职责**：
- 执行四阶段工作流（Python 原生 async/await）
- 管理状态转换和累积
- 处理阶段启用/禁用逻辑
- 协调并行和串行执行

**核心调度函数**：

```python
# scheduler/workflow_scheduler.py

from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.messages import HumanMessage, SystemMessage
import asyncio

async def run_trading_analysis(symbol: str, config: dict, progress_callback=None):
    """
    执行四阶段股票分析工作流

    Args:
        symbol: 股票代码
        config: 分析配置（包含智能体配置、模型配置等）
        progress_callback: 进度回调函数

    Returns:
        AnalysisState: 最终分析状态
    """
    # 初始化状态
    state = AnalysisState(
        symbol=symbol,
        user_id=config["user_id"],
        task_id=config["task_id"]
    )

    # 初始化模型
    model = ChatOpenAI(
        model=config.get("model", "gpt-4o"),
        temperature=config.get("temperature", 0.7)
    )

    # Phase 1: 并行分析师团队
    if config.get("enable_phase1", True):
        state.current_phase = 1
        if progress_callback:
            await progress_callback("phase1_start", {"phase": 1})

        phase1_results = await run_phase1_analysts(
            state, model, config["phase1_analysts"]
        )
        state.analyst_reports = phase1_results

        if progress_callback:
            await progress_callback("phase1_complete", phase1_results)

    # Phase 2: 串行观点辩论
    if config.get("enable_phase2", True):
        state.current_phase = 2
        if progress_callback:
            await progress_callback("phase2_start", {"phase": 2})

        phase2_results = await run_phase2_debate(
            state, model, config["phase2_debaters"]
        )
        state.debate_turns = phase2_results

        if progress_callback:
            await progress_callback("phase2_complete", phase2_results)

    # Phase 3: 并行风险评估
    if config.get("enable_phase3", True):
        state.current_phase = 3
        if progress_callback:
            await progress_callback("phase3_start", {"phase": 3})

        phase3_results = await run_phase3_risk(
            state, model, config["phase3_assessors"]
        )
        state.risk_assessments = phase3_results

        if progress_callback:
            await progress_callback("phase3_complete", phase3_results)

    # Phase 4: 综合报告
    if config.get("enable_phase4", True):
        state.current_phase = 4
        if progress_callback:
            await progress_callback("phase4_start", {"phase": 4})

        final_report = await run_phase4_report(state, model)
        state.final_report = final_report

        if progress_callback:
            await progress_callback("phase4_complete", final_report)

    state.status = "completed"
    return state
```

### 3.4 阶段函数 (Phase Functions)

**Phase 1: 分析师团队（并行）**

```python
# phases/phase1.py

async def run_phase1_analysts(state: AnalysisState, model, analyst_configs: list):
    """
    并行执行所有分析师

    使用 asyncio.gather() 实现真正的并行
    """
    async def run_single_analyst(config):
        # 创建提示词
        prompt = ChatPromptTemplate.from_messages([
            ("system", config["system_prompt"]),
            ("human", "请分析股票：{symbol}")
        ])

        # 如果配置了工具，绑定工具
        if config.get("tools"):
            model_with_tools = model.bind_tools(config["tools"])
        else:
            model_with_tools = model

        # 调用模型
        chain = prompt | model_with_tools
        result = await chain.ainvoke({"symbol": state.symbol})

        return {
            "analyst": config["name"],
            "report": result.content,
            "role": config["role"]
        }

    # 并行执行所有分析师
    tasks = [run_single_analyst(config) for config in analyst_configs]
    results = await asyncio.gather(*tasks, return_exceptions=True)

    # 处理异常
    analyst_reports = []
    for i, result in enumerate(results):
        if isinstance(result, Exception):
            analyst_reports.append({
                "analyst": analyst_configs[i]["name"],
                "error": str(result)
            })
        else:
            analyst_reports.append(result)

    return analyst_reports
```

**Phase 2: 观点辩论（串行）**

```python
# phases/phase2.py

async def run_phase2_debate(state: AnalysisState, model, debater_configs: list):
    """
    串行执行观点辩论

    看涨和看跌辩手交替发言，多轮辩论
    """
    debate_turns = []
    rounds = debater_configs[0].get("rounds", 2)

    for round_num in range(rounds):
        # 看涨辩手
        bull_prompt = ChatPromptTemplate.from_messages([
            ("system", debater_configs[0]["bull_prompt"]),
            ("human", "基于以下分析报告，阐述看涨观点：\n{reports}")
        ])
        bull_chain = bull_prompt | model
        bull_response = await bull_chain.ainvoke({
            "reports": format_reports(state.analyst_reports),
            "previous_turns": format_debate_turns(debate_turns)
        })

        debate_turns.append({
            "role": "bull",
            "content": bull_response.content,
            "round": round_num + 1
        })

        # 看跌辩手
        bear_prompt = ChatPromptTemplate.from_messages([
            ("system", debater_configs[0]["bear_prompt"]),
            ("human", "基于以下分析报告和看涨观点，阐述看跌观点：\n{reports}\n\n{bull_view}")
        ])
        bear_chain = bear_prompt | model
        bear_response = await bear_chain.ainvoke({
            "reports": format_reports(state.analyst_reports),
            "bull_view": bull_response.content
        })

        debate_turns.append({
            "role": "bear",
            "content": bear_response.content,
            "round": round_num + 1
        })

    return debate_turns
```

**Phase 3: 风险评估（并行）**

```python
# phases/phase3.py

async def run_phase3_risk(state: AnalysisState, model, assessor_configs: list):
    """
    并行执行风险评估
    """
    async def run_single_assessor(config):
        prompt = ChatPromptTemplate.from_messages([
            ("system", config["system_prompt"]),
            ("human", "基于以下辩论结果，从{perspective}角度评估风险：\n{debate}")
        ])
        chain = prompt | model
        result = await chain.ainvoke({
            "debate": format_debate_turns(state.debate_turns),
            "perspective": config["perspective"]
        })
        return {
            "assessor": config["name"],
            "assessment": result.content,
            "perspective": config["perspective"]
        }

    # 并行执行
    tasks = [run_single_assessor(config) for config in assessor_configs]
    results = await asyncio.gather(*tasks, return_exceptions=True)

    return [r for r in results if not isinstance(r, Exception)]
```

### 3.5 WebSocket 推送管理器

**职责**：
- 管理 WebSocket 连接
- 推送实时进度事件
- 处理连接认证和权限
- 连接池管理（最多 5 个/用户）

**断线重连机制**：
- **心跳检测**：每 30 秒发送一次心跳包
- **自动重连**：连接断开后自动重连（指数退避策略）
- **状态恢复**：重连成功后自动同步任务最新状态
- **最大重试**：最多重连 10 次，超过后提示用户手动刷新

## 六、技术选型

### 6.1 工作流实现

**Python 原生 async/await + LangChain 基础组件**

选择理由：
- **简单直接**：无需学习复杂的编排框架
- **易于调试**：Python 原生代码，断点调试友好
- **灵活可控**：完全掌控执行流程
- **高效并行**：asyncio.gather() 实现真正的并行处理

### 6.2 LangChain 组件使用

| 组件 | 用途 | 说明 |
|------|------|------|
| `ChatOpenAI` | LLM 调用 | 直接调用 `.invoke()` 或 `.ainvoke()` |
| `ChatPromptTemplate` | 提示词管理 | 支持模板变量和消息格式 |
| `.bind_tools()` | 工具绑定 | 将 MCP 工具绑定到模型 |
| `StrOutputParser` | 输出解析 | 提取文本内容 |

### 6.3 数据存储

| 数据类型 | 存储选择 | 理由 |
|----------|----------|------|
| 任务状态 | MongoDB | 持久化、灵活查询 |
