## 一、数据库架构概述

### 1.1 数据库选型

| 属性 | 说明 |
|------|------|
| **数据库** | MongoDB |
| **版本** | 5.0+ |
| **驱动** | Motor (异步 Python) |

### 1.2 集合列表

| 集合名 | 用途 | 数据量级 | 保留策略 |
|--------|------|----------|----------|
| `ai_models` | AI 模型配置 | 少 (10-100) | 永久 |
| `mcp_servers` | MCP 服务器配置 | 少 (5-50) | 永久 |
| `agent_configs` | 智能体配置 | 少 (用户数) | 永久 |
| `analysis_tasks` | 分析任务 | 中 (千-万级) | 永久 |
| `analysis_reports` | 分析报告 | 中 (千-万级) | 30天后归档 |
| `archived_reports` | 归档报告 | 大 (万-百万级) | 90天后删除 |
| `agent_traces` | 智能体执行跟踪 | 大 (百万级) | 7天后删除 |
| `trading_agents_settings` | 用户设置 | 少 (用户数) | 永久 |
| `alerts` | 告警记录 | 中 (千-万级) | 30天后删除 |

### 1.3 数据隔离原则

**强制用户隔离**：所有用户相关数据必须包含 `user_id` 字段，查询时强制过滤。

| 集合 | user_id 字段 | 说明 |
|------|--------------|------|
| `analysis_tasks` | ✓ | 强制索引 |
| `analysis_reports` | ✓ | 强制索引 |
| `agent_configs` | ✓ | 唯一约束 |
| `trading_agents_settings` | ✓ | 唯一约束 |
| `ai_models` | ✓ (owner_id) | 所有者隔离 |
| `mcp_servers` | ✓ (owner_id) | 所有者隔离 |
| `agent_traces` | × | 通过 task_id 关联 |
| `archived_reports` | ✓ | 归档后保留 |
| `alerts` | × | 系统级别 |

## 二、集合设计

### 2.1 AI 模型配置 (ai_models)

**用途**：存储用户配置的 AI 模型（OpenAI、Anthropic、本地模型等）

**文档结构**：

```javascript
{
  "_id": ObjectId("..."),
  "owner_id": "user123",           // 所有者用户ID (系统模型为 "system")
  "name": "Claude Sonnet 4.5",     // 模型显示名称
  "platform_type": "preset",       // 平台类型: preset/custom
  "platform_name": "anthropic",    // 预设平台名称
  "api_base_url": "https://api.anthropic.com/v1",
  "api_key": "sk-ant-xxxxx",       // 加密存储
  "model_id": "claude-sonnet-4-20250514",
  "max_concurrency": 5,            // 最大并发数
  "task_concurrency": 2,           // 任务级并发
  "batch_concurrency": 1,          // 批量级并发
  "timeout_seconds": 60,           // 超时时间
  "temperature": 0.7,              // 温度参数
  "thinking_enabled": true,        // 是否启用思考模式
  "thinking_mode": "auto",         // 思考模式: auto/extended
  "enabled": true,                 // 是否启用
  "is_system": false,              // 是否为系统模型
  "price_config": {                // 价格配置 (可选)
    "input_price": 3.00,           // 元/百万tokens
    "output_price": 15.00,
    "thinking_price": 3.75
  },
  "created_at": ISODate("2026-01-14T10:00:00Z"),
  "updated_at": ISODate("2026-01-14T10:00:00Z")
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 索引 | 说明 |
|------|------|------|------|------|
| `_id` | ObjectId | 是 | 主键 | 文档ID |
| `owner_id` | string | 是 | ✓ | 所有者用户ID |
| `name` | string | 是 | - | 模型显示名称 |
| `platform_type` | string | 是 | - | 平台类型 |
| `platform_name` | string | 否 | - | 预设平台名称 |
| `api_base_url` | string | 是 | - | API 地址 |
| `api_key` | string | 是 | - | API 密钥 (加密) |
| `model_id` | string | 是 | - | 模型标识符 |
| `max_concurrency` | int | 是 | - | 最大并发数 |
| `task_concurrency` | int | 是 | - | 任务级并发 |
| `batch_concurrency` | int | 是 | - | 批量级并发 |
| `timeout_seconds` | int | 是 | - | 超时时间 |
| `temperature` | float | 否 | - | 温度参数 |
| `thinking_enabled` | bool | 否 | - | 启用思考模式 |
| `thinking_mode` | string | 否 | - | 思考模式 |
| `enabled` | bool | 是 | ✓ | 是否启用 |
| `is_system` | bool | 是 | ✓ | 是否系统模型 |
| `price_config` | object | 否 | - | 价格配置 |
| `created_at` | datetime | 是 | ✓ | 创建时间 |
| `updated_at` | datetime | 是 | - | 更新时间 |

**索引**：
- `idx_owner_id`: `{ "owner_id": 1 }`
- `idx_is_system`: `{ "is_system": 1 }`
- `idx_enabled`: `{ "enabled": 1 }`
- `idx_created_at`: `{ "created_at": -1 }`
- `idx_owner_enabled`: `{ "owner_id": 1, "enabled": 1 }`

### 2.3 智能体配置 (agent_configs)

**用途**：存储用户的智能体个人配置（公共配置存储在 YAML 文件中）

**配置层级**：
1. **默认配置**: `backend/modules/trading_agents/config/agents/agents_default.yaml`（项目内置，永不修改）
2. **公共配置**: `backend/modules/trading_agents/config/agents/agents_public.yaml`（管理员可修改）
3. **用户个人配置**: MongoDB `agent_configs` 集合（用户自定义，优先级最高）

**文档结构**：

```javascript
{
  "_id": ObjectId("..."),
  "user_id": "user123",           // 用户ID (唯一)
  "is_customized": true,           // 是否已自定义（True=使用个人配置，False=使用公共配置YAML）
  "config": {                      // 完整的四阶段配置
    "phase1": {
      "enabled": true,
      "max_turns": 3,
      "max_concurrency": 3,
      "agents": [
        {
          "slug": "technical",
          "name": "技术分析师",
          "description": "进行技术分析，识别趋势和形态",
          "system_prompt": "你是技术分析师...",        // 系统提示词
          "mcp_servers": ["财经数据"],                // 启用的 MCP 服务器名称列表
          "local_tools": ["get_stock_quotes"],         // 启用的本地工具名称列表
          "enabled": true
        }
      ]
    },
    "phase2": {
      "enabled": true,
      "debate_rounds": 2,
      "agents": [
        {
          "slug": "moderator",
          "name": "主持人",
          "description": "协调辩论过程",
          "system_prompt": "你是辩论主持人...",
          "enabled": true
        }
      ]
    },
    "phase3": {
      "enabled": true,
      "execution_mode": "serial",
      "agents": [...]
    },
    "phase4": {
      "enabled": true,
      "agents": [...]
    }
  },
  "created_at": ISODate("2026-01-14T10:00:00Z"),
  "updated_at": ISODate("2026-01-14T10:00:00Z")
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 索引 | 说明 |
|------|------|------|------|------|
| `_id` | ObjectId | 是 | 主键 | 文档ID |
| `user_id` | string | 是 | ✓(unique) | 用户ID |
| `is_customized` | bool | 是 | ✓ | 是否已自定义 |
| `config` | object | 是 | - | 完整的四阶段配置 |
| `config.phase1` | object | 是 | - | Phase 1 配置 |
| `config.phase2` | object | 是 | - | Phase 2 配置 |
| `config.phase3` | object | 是 | - | Phase 3 配置 |
| `config.phase4` | object | 是 | - | Phase 4 配置 |
| `created_at` | datetime | 是 | ✓ | 创建时间 |
| `updated_at` | datetime | 是 | ✓ | 更新时间 |

**阶段配置子结构**：

```javascript
{
  "enabled": true,                 // 是否启用该阶段
  "max_turns": 3,                  // 最大轮次 (Phase 1)
  "max_concurrency": 3,            // 最大并发数 (Phase 1)
  "debate_rounds": 2,              // 辩论轮数 (Phase 2)
  "execution_mode": "serial",      // 执行模式 (Phase 3: serial/parallel)
  "agents": [                      // 智能体列表
    {
      "slug": "technical",         // 唯一标识符
      "name": "技术分析师",         // 显示名称
      "description": "进行技术分析，识别趋势和形态",
      "system_prompt": "...",      // 系统提示词
      "mcp_servers": [...],        // MCP 服务器名称列表
      "local_tools": [...],        // 本地工具名称列表
      "enabled": true              // 是否启用该智能体
    }
  ]
}
```

**索引**：
- `idx_user_id`: `{ "user_id": 1 }` (unique)
- `idx_is_customized`: `{ "is_customized": 1 }`
- `idx_created_at`: `{ "created_at": -1 }`
- `idx_updated_at`: `{ "updated_at": -1 }`

**读取优先级**：

```
1. 数据库有记录且 is_customized=true → 使用数据库个人配置
2. 数据库无记录或 is_customized=false → 读取 agents_public.yaml
3. agents_public.yaml 不存在 → 读取 agents_default.yaml（兜底）
```

**用户操作说明**：

- **修改配置**:
  - 数据库无记录 → 创建新记录，设置 is_customized=true
  - 数据库有记录 → 更新记录，设置 is_customized=true
  - 存储完整的四阶段配置（不区分改了什么）

- **恢复公共配置**:
  - 删除数据库中的用户记录
  - 后续读取将使用 agents_public.yaml

- **导出个人配置**:
  - 下载用户的数据库记录（YAML 格式）

- **导入个人配置**:
  - 上传 YAML 文件 → 创建/更新数据库记录

```javascript
{
  "_id": ObjectId("..."),
  "task_id": ObjectId("..."),     // 关联任务ID
  "user_id": "user123",
  "stock_code": "600000",
  "stock_name": "浦发银行",
  "market": "a_share",
```

## 五、数据生命周期

### 5.1 数据保留策略

| 数据类型 | 保留期限 | 清理方式 | 清理任务 |
|----------|----------|----------|----------|
| 任务记录 (`analysis_tasks`) | 永久 | 不清理 | - |
| 活跃报告 (`analysis_reports`) | 30 天 | 归档到 `archived_reports` | 每日归档任务 |
| 归档报告 (`archived_reports`) | 90 天 | 删除文档 | 每日清理任务 |
| 执行跟踪 (`agent_traces`) | 7 天 | 删除文档 | 每日清理任务 |
| 告警记录 (`alerts`) | 30 天 | 删除文档 | 每日清理任务 |

### 5.2 数据归档流程

**报告归档** (30 天)：
```
1. 查询 `analysis_reports` 中 `created_at < 30天前` 且 `archived = false`
2. 复制到 `archived_reports`，添加归档元数据
3. 标记原记录 `archived = true`
4. 可选：从原记录删除 `report_content` 大字段以节省空间
```

**归档报告清理** (90 天)：
```
1. 查询 `archived_reports` 中 `archived_at < 90天前`
2. 删除文档
```

### 5.3 任务过期处理

**任务过期** (24 小时)：
```
1. 查询 `status = pending` 且 `created_at < 24小时前` 的任务
2. 更新状态为 `expired`
3. 设置 `expired_at` 时间戳
```

### 5.4 定时任务

| 任务名称 | 执行频率 | 处理逻辑 |
|----------|----------|----------|
| `archive_reports` | 每日 02:00 | 归档 30 天前的报告 |
| `cleanup_archived_reports` | 每日 03:00 | 删除 90 天前的归档报告 |
| `cleanup_agent_traces` | 每日 04:00 | 删除 7 天前的执行跟踪 |
| `cleanup_alerts` | 每日 05:00 | 删除 30 天前的告警 |
| `expire_pending_tasks` | 每小时 | 过期 24 小时未执行的任务 |
