## 一、页面结构总览

### 1.1 全局导航结构

TradingAgents 模块在前端全局导航中的位置：

```text
全局设置
├── 用户管理 [adminOnly]
├── 系统设置 [adminOnly]
├── MCP 服务器管理
├── AI 模型管理
└── TradingAgent 设置
    ├── 智能体配置
    └── 分析设置
```

### 1.2 模块内导航结构

```text
TradingAgents 模块
├── 分析
│   ├── 单股分析
│   ├── 批量分析
│   └── 分析详情
├── 任务中心
└── 历史对比
```

### 1.3 模块说明

| 页面 | 所属模块 | 说明 |
|------|---------|------|
| 分析页面 | TradingAgents | 股票分析功能核心页面 |
| 任务中心 | TradingAgents | 任务管理与追踪 |
| 历史对比 | TradingAgents | 历史分析对比与准确率统计 |
| 智能体配置 | TradingAgents | 四阶段智能体配置 |
| 分析设置 | TradingAgents | 分析参数设置 |

**不属于本模块的页面**：
- AI 模型管理 → 属于 `core/ai` 核心模块
- MCP 服务器管理 → 属于 `modules/mcp` 模块
- 管理员页面 → 系统级管理功能

**核心功能**:
1. **状态筛选**
   - 标签式筛选（全部/进行中/已完成/失败）
   - 徽章显示各状态数量

2. **搜索与高级筛选**
   - 股票代码模糊搜索
   - 推荐结果筛选
   - 风险等级筛选
   - 日期范围筛选

3. **批量操作**
   - 全选/取消全选
   - 批量删除
   - 清空指定状态任务

4. **任务列表**
   - 分页加载（每页 50 条）
   - 实时状态更新
   - 快捷操作（查看详情、重试、删除）

5. **任务操作**
   - 查看详情 → 跳转到分析详情页
   - 重试 → 使用原配置创建新任务
   - 删除 → 删除任务（需确认）

6. **批量任务状态汇总**（新增）
   - 显示最近执行的批量任务状态
   - 展示总数、成功、失败、进行中数量
   - 点击查看详情可展开该批量任务的所有子任务
   - 失败的子任务可单独重试（自动重试 3 次后标记为失败）

**批量任务失败处理**：

```
重试策略：
- 第 1 次：立即重试
- 第 2 次：等待 5 秒后重试
- 第 3 次：等待 15 秒后重试
- 3 次全部失败后标记为失败状态
- 单个任务失败不影响其他任务继续执行
```

**列表项字段**:

| 字段 | 说明 | 格式 |
|------|------|------|
| stock_code | 股票代码 | 600000 |
| stock_name | 股票名称 | 浦发银行 |
| status | 状态 | 带颜色的图标 |
| recommendation | 推荐结果 | 买入/持有/卖出 |
| progress | 进度 | 百分比 |
| token_usage | Token 用量 | 数量 + 估算成本 |
| created_at | 创建时间 | 2026-01-14 10:00 |

## 八、WebSocket 断线重连机制

### 8.1 前端实现要点

**文件路径**: `frontend/src/modules/trading_agents/composables/useWebSocket.ts`

**重连策略**:

```text
第1次：立即重连
第2次：等待 2 秒
第3次：等待 5 秒
第4次：等待 10 秒
第5次及以后：等待 30 秒
最多重连：10 次
```

**核心功能**:

1. **心跳检测**: 每 30 秒发送一次心跳包
2. **自动重连**: 连接断开后自动重连
3. **状态恢复**: 重连成功后自动同步任务最新状态
4. **最大重试**: 最多重连 10 次，超过后提示用户手动刷新

**状态管理**:

| 状态 | 说明 |
|------|------|
| connecting | 正在连接 |
| connected | 已连接 |
| disconnected | 已断开 |
| reconnecting | 重连中 |
| failed | 连接失败（超过最大重试次数） |

**用户提示**:

- 连接断开时显示顶部提示条
- 重连中显示进度和剩余次数
- 超过最大重试次数后显示"手动刷新"按钮
