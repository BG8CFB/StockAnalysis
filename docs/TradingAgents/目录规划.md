## 一、目录规划原则

### 1.1 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                    前端 (Vue 3)                            │
├─────────────────────────────────────────────────────────────┤
│                    后端 API 层                              │
├─────────────────────────────────────────────────────────────┤
│                    服务层 (Services)                        │
├─────────────────────────────────────────────────────────────┤
│                    管理层 (Managers)                        │
├─────────────────────────────────────────────────────────────┤
│                    调度层 (Schedulers)                      │
├─────────────────────────────────────────────────────────────┤
│                    阶段层 (Phases)                          │
├─────────────────────────────────────────────────────────────┤
│                    推送层 (Pushers)                         │
├─────────────────────────────────────────────────────────────┤
│                    基础设施层 (Infrastructure)               │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 组织原则

1. **按功能模块划分**: 相关代码放在一起
2. **高内聚低耦合**: 层级清晰，职责单一
3. **前后端分离**: 独立的目录结构
4. **配置集中管理**: 统一的配置目录

## 二、后端目录结构

### 2.1 完整目录树

```
backend/modules/trading_agents/
├── __init__.py
├── admin_api.py              # 管理员 API 路由
├── schemas.py                # Pydantic 数据模型
├── exceptions.py             # 自定义异常
│
├── api/                      # API 层 - 对外接口
│   ├── __init__.py
│   ├── tasks.py              # 任务管理 API
│   └── reports.py            # 报告查询 API
│
├── services/                 # 服务层 - 业务逻辑
│   ├── __init__.py
│   ├── report_service.py     # 报告服务
│   ├── agent_config_service.py  # 智能体配置服务
│   └── settings_service.py   # 用户设置服务
│
├── manager/                  # 管理层 - 任务管理核心
│   ├── __init__.py
│   ├── task_manager.py       # 任务管理器
│   ├── concurrency_controller.py  # 并发控制器
│   ├── batch_manager.py      # 批量任务管理器
│   ├── task_queue.py         # 任务队列
│   ├── task_expiry.py        # 任务过期处理
│   └── task_manager_restore.py  # 任务恢复
│
├── scheduler/                # 调度层 - 工作流调度
│   ├── __init__.py
│   ├── workflow_scheduler.py # 工作流调度器
│   └── state.py              # 状态管理
│
├── phases/                   # 阶层层 - 四阶段执行
│   ├── __init__.py
│   ├── phase1/
│   │   ├── __init__.py
│   │   └── runner.py         # Phase 1 运行器（并行分析）
│   ├── phase2/
│   │   ├── __init__.py
│   │   └── runner.py         # Phase 2 运行器（观点辩论）
│   ├── phase3/
│   │   ├── __init__.py
│   │   └── runner.py         # Phase 3 运行器（风险评估）
│   └── phase4/
│       ├── __init__.py
│       └── runner.py         # Phase 4 运行器（综合报告）
│
├── pusher/                   # 推送层 - 实时通信
│   ├── __init__.py
│   ├── events.py             # 事件定义
│   └── manager.py            # WebSocket 管理器
│
├── websocket/                # WebSocket 路由
│   ├── __init__.py
│   └── routes.py             # WebSocket 端点
│
├── tools/                    # 工具模块 - 本地工具
│   ├── __init__.py
│   ├── mcp_tool_filter.py    # MCP 工具过滤器
│   └── loop_detector.py      # 工具循环检测
│
├── infra/                    # 基础设施 - 底层支持
│   ├── __init__.py
│   ├── database.py           # 数据库操作
│   └── alerts.py             # 告警处理
│
├── models/                   # 数据模型 - 工作流状态
│   ├── __init__.py
│   └── state.py              # 工作流状态模型
│
└── config/                   # 配置管理 - 智能体配置
    ├── __init__.py
    ├── agents/               # 智能体配置文件
    │   ├── agents_default.yaml     # 默认配置（项目内置）
    │   └── agents_public.yaml      # 公共配置（管理员可修改）
    ├── loader.py             # 配置加载器
    └── service.py            # 配置服务
```

### 2.2 目录功能说明

| 目录 | 职责 | 关键文件 |
|------|------|----------|
| `api/` | 对外接口层 | 任务创建、查询、取消 API |
| `services/` | 业务服务层 | 报告服务、配置服务 |
| `manager/` | 任务管理层 | 任务生命周期管理 |
| `scheduler/` | 工作流调度层 | 四阶段编排执行 |
| `phases/` | 阶段执行层 | 各阶段独立运行器 |
| `pusher/` | 实时推送层 | WebSocket 事件推送 |
| `tools/` | 工具模块层 | MCP 工具集成 |
| `infra/` | 基础设施层 | 数据库、告警 |
| `models/` | 数据模型层 | 状态模型定义 |
| `config/` | 配置管理层 | 智能体配置文件 |

## 三、前端目录结构

│   │   ├── AgentStatusCard.vue         # 智能体状态卡片

│   │   ├── ReportCard.vue              # 报告卡片

│   │   └── StreamingReport.vue         # 流式报告
│   ├── settings/             # 设置相关组件
│   │   └── PhaseConfigPanel.vue        # 阶段配置面板
│   └── common/               # 通用组件

│   ├── useWebSocket.ts       # WebSocket 连接管理
│   ├── useTaskProgress.ts    # 任务进度追踪

│   ├── tasks.ts              # 任务相关 API

│   ├── tasks.ts              # 任务状态
│   ├── config.ts             # 配置状态

│   ├── report.ts             # 报告类型
│   ├── config.ts             # 配置类型
│   └── websocket.ts          # WebSocket 类型

### 3.2 页面路由映射

| 路由 | 组件 | 功能 |
|------|------|------|
| `/trading-agents/analysis/single` | SingleAnalysisView | 单股分析 |
| `/trading-agents/analysis/batch` | BatchAnalysisView | 批量分析 |
| `/trading-agents/analysis/:taskId` | AnalysisDetailView | 分析详情 |
| `/trading-agents/tasks` | TaskCenterView | 任务中心 |
| `/trading-agents/compare` | HistoryCompareView | 历史对比 |
| `/settings/trading-agents/agent-config` | AgentConfigView | 智能体配置 |
| `/settings/trading-agents/analysis` | AnalysisSettingsView | 分析设置 |

### 3.3 目录功能说明

| 目录 | 职责 | 关键内容 |
|------|------|----------|
| `views/` | 页面视图 | 分析、任务、设置、对比页面 |
| `components/` | 共享组件 | 可复用的 Vue 组件 |
| `composables/` | 组合式函数 | 可复用的逻辑函数 |
| `api/` | API 调用 | 后端接口封装 |
| `stores/` | 状态管理 | Pinia 状态存储 |
| `types/` | 类型定义 | TypeScript 类型 |
| `utils/` | 工具函数 | 通用工具方法 |

## 五、配置文件结构

### 5.1 后端配置文件

```
backend/modules/trading_agents/config/agents/
├── agents_default.yaml       # 默认配置（项目内置，永不修改）
└── agents_public.yaml        # 公共配置（管理员可修改）
```

### 5.2 配置文件说明

| 文件 | 属性 | 作用 |
|------|------|------|
| `agents_default.yaml` | 项目内置 | 作为备份和兜底 |
| `agents_public.yaml` | 管理员可修改 | 系统默认配置 |

### 5.3 配置结构

```yaml
# 四阶段配置结构
phase1:                      # Phase 1: 分析师团队（完全可配置）
  enabled: bool
  max_concurrency: int
  agents:
    - slug: str
      name: str
      description: str
      system_prompt: str
      mcp_servers: [str]
      local_tools: [str]
      enabled: bool

phase2:                      # Phase 2: 观点辩论（固定智能体）
  enabled: bool
  debate_rounds: int
  agents: [...]

phase3:                      # Phase 3: 风险评估（固定智能体）
  enabled: bool
  execution_mode: str
  agents: [...]

phase4:                      # Phase 4: 综合报告（固定智能体）
  enabled: bool
  agents: [...]
```

## 六、数据流向图

### 6.1 前端数据流

```
用户操作
    ↓
组件 (Components)
    ↓
Composables (逻辑处理)
    ↓
API (api/*.ts)
    ↓
Stores (状态更新)
    ↓
组件重新渲染
```

### 6.2 后端数据流

```
API 请求
    ↓
api/*.py (路由层)
    ↓
services/*.py (服务层)
    ↓
manager/*.py (管理层)
    ↓
scheduler/*.py (调度层)
    ↓
phases/*/runner.py (执行层)
    ↓
pusher/*.py (推送层)
    ↓
WebSocket 实时推送
```
