## 一、配置层级结构

### 1.1 三层配置架构

```
┌─────────────────────────────────────────────────────────────┐
│                      配置层级                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 默认配置（Default Config）                               │
│     ├── 文件: agents_default.yaml                           │
│     ├── 位置: backend/modules/trading_agents/config/agents/ │
│     ├── 属性: 项目内置，永不修改                             │
│     └── 作用: 作为备份和兜底                                 │
│                                                             │
│  2. 公共配置（Public Config）                                │
│     ├── 文件: agents_public.yaml                            │
│     ├── 位置: backend/modules/trading_agents/config/agents/ │
│     ├── 属性: 管理员可修改，所有用户共享                     │
│     └── 作用: 系统默认配置                                   │
│                                                             │
│  3. 用户个人配置（User Config）                              │
│     ├── 存储: MongoDB agent_configs 集合                    │
│     ├── 属性: 用户自定义，优先级最高                          │
│     └── 作用: 个人定制配置                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 配置文件路径

```
backend/modules/trading_agents/config/agents/
├── agents_default.yaml     # 默认配置（项目内置，不修改）
└── agents_public.yaml      # 公共配置（管理员修改）
```

### 1.3 数据库集合

```
MongoDB Collections
└── agent_configs           # 用户个人配置
```

## 二、配置文件管理

### 2.1 默认配置 (agents_default.yaml)

**属性**：项目内置配置，永不修改，作为备份

**初始化时机**：
- 项目部署时从源码复制
- 系统启动时检查是否存在

**内容示例**：

```yaml
phase1:
  enabled: true
  max_turns: 3
  max_concurrency: 3
  agents:
    - slug: technical_analyst
      name: 技术分析师
      description: 进行技术分析，识别趋势和形态
      system_prompt: |
        你是一名专业的股票技术分析师...
      mcp_servers:
        - 财经数据
      local_tools:
        - get_stock_quotes
      enabled: true
    # ... 更多智能体

phase2:
  enabled: true
  debate_rounds: 2
  agents:
    - slug: moderator
      name: 主持人
      description: 协调辩论过程
      system_prompt: |
        你是辩论主持人...
      enabled: true
    # ... 更多智能体

phase3:
  enabled: true
  execution_mode: serial
  agents: [...]

phase4:
  enabled: true
  agents: [...]
```

### 2.2 公共配置 (agents_public.yaml)

**属性**：管理员可修改，所有用户共享

**初始化时机**：
- 系统启动时检查是否存在
- 不存在则从 `agents_default.yaml` 复制

**管理操作**：
- 管理员通过 API 直接修改文件
- 支持导出/导入 YAML 文件
- 支持恢复为默认配置

### 2.3 配置文件格式

**YAML 结构**：

```yaml
# 阶段配置
phase1:
  enabled: bool              # 是否启用
  max_turns: int             # 最大轮次
  max_concurrency: int       # 最大并发
  agents:
    - slug: str              # 唯一标识
      name: str              # 显示名称
      description: str       # 描述
      system_prompt: str     # 系统提示词
      mcp_servers: [str]     # MCP 服务器列表
      local_tools: [str]     # 本地工具列表
      enabled: bool          # 是否启用

phase2:
  enabled: bool
  debate_rounds: int         # 辩论轮数
  agents: [...]

phase3:
  enabled: bool
  execution_mode: str       # 执行模式
  agents: [...]

phase4:
  enabled: bool
  agents: [...]
```

## 三、数据库配置管理

### 3.1 集合结构

**集合名**: `agent_configs`

**文档结构**：

```javascript
{
  "_id": ObjectId("..."),
  "user_id": "user123",
  "is_customized": true,           // 是否已自定义
  "config": {
    "phase1": { ... },
    "phase2": { ... },
    "phase3": { ... },
    "phase4": { ... }
  },
  "created_at": ISODate("2026-01-14T10:00:00Z"),
  "updated_at": ISODate("2026-01-14T10:00:00Z")
}
```

### 3.2 字段说明

| 字段 | 类型 | 必填 | 索引 | 说明 |
|------|------|------|------|------|
| `_id` | ObjectId | 是 | 主键 | 文档ID |
| `user_id` | string | 是 | ✓(unique) | 用户ID |
| `is_customized` | bool | 是 | ✓ | 是否已自定义 |
| `config` | object | 是 | - | 完整的四阶段配置 |
| `created_at` | datetime | 是 | ✓ | 创建时间 |
| `updated_at` | datetime | 是 | ✓ | 更新时间 |

### 3.3 索引

```javascript
// 唯一索引
db.agent_configs.createIndex({ "user_id": 1 }, { unique: true })

// 普通索引
db.agent_configs.createIndex({ "is_customized": 1 })
db.agent_configs.createIndex({ "created_at": -1 })
db.agent_configs.createIndex({ "updated_at": -1 })
```

## 四、配置读取优先级

### 4.1 读取流程

```
┌─────────────────────────────────────────────────────────────┐
│                    用户配置读取流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 检查数据库是否有用户个人配置记录                          │
│     └── 查询: { user_id: "user123", is_customized: true }    │
│         有记录 → 返回数据库配置 (优先级最高)                   │
│                                                             │
│  2. 数据库无记录 → 读取公共配置 YAML 文件                     │
│     └── 文件: agents_public.yaml                            │
│         文件存在 → 返回文件内容                               │
│         文件不存在 → 继续下一步                               │
│                                                             │
│  3. 读取默认配置 YAML 文件（兜底）                            │
│     └── 文件: agents_default.yaml                           │
│         返回默认配置内容                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 读取逻辑伪代码

```python
async def get_agent_config(user_id: str) -> dict:
    # 1. 检查数据库个人配置
    db_config = await db.agent_configs.find_one({
        "user_id": user_id,
        "is_customized": True
    })
    if db_config:
        return db_config["config"]

    # 2. 读取公共配置 YAML
    public_config_path = "config/agents/agents_public.yaml"
    if os.path.exists(public_config_path):
        return load_yaml(public_config_path)

    # 3. 读取默认配置 YAML（兜底）
    default_config_path = "config/agents/agents_default.yaml"
    return load_yaml(default_config_path)
```

## 五、管理员操作

### 5.1 公共配置管理 API

#### 5.1.1 读取公共配置

**接口**：`GET /api/trading-agents/admin/public-config`

**响应**：

```javascript
{
  "success": true,
  "data": {
    "phase1": { ... },
    "phase2": { ... },
    "phase3": { ... },
    "phase4": { ... }
  },
  "source": "yaml_file"  // 数据来源标识
}
```

#### 5.1.2 更新公共配置

**接口**：`PUT /api/trading-agents/admin/public-config`

**请求体**：

```javascript
{
  "phase1": { ... },
  "phase2": { ... },
  "phase3": { ... },
  "phase4": { ... }
}
```

**逻辑**：
- 直接覆盖写入 `agents_public.yaml`
- 原子操作，避免并发写入冲突

#### 5.1.3 导出公共配置

**接口**：`GET /api/trading-agents/admin/public-config/export`

**响应**：YAML 文件下载

#### 5.1.4 导入公共配置

**接口**：`POST /api/trading-agents/admin/public-config/import`

**请求**：`multipart/form-data` 上传 YAML 文件

**逻辑**：
- 验证 YAML 格式
- 验证配置结构
- 直接覆盖 `agents_public.yaml`

#### 5.1.5 恢复默认配置

**接口**：`POST /api/trading-agents/admin/public-config/reset`

**逻辑**：
- 用 `agents_default.yaml` 覆盖 `agents_public.yaml`
- 等同于"恢复出厂设置"

### 5.2 项目初始化流程

**系统启动时**：

```python
async def init_agent_configs():
    public_config_path = "config/agents/agents_public.yaml"
    default_config_path = "config/agents/agents_default.yaml"

    # 检查公共配置是否存在
    if not os.path.exists(public_config_path):
        # 复制默认配置
        shutil.copy(default_config_path, public_config_path)
        logger.info("Created public config from default")
```

## 六、用户操作

### 6.1 读取配置

**接口**：`GET /api/trading-agents/agent-config`

**响应**：

```javascript
{
  "success": true,
  "data": {
    "config": { ... },           // 完整的四阶段配置
    "source": "personal",         // personal/public/default
    "is_customized": true         // 是否使用个人配置
  }
}
```

### 6.2 修改配置

**接口**：`PUT /api/trading-agents/agent-config`

**请求体**：

```javascript
{
  "phase1": { ... },
  "phase2": { ... },
  "phase3": { ... },
  "phase4": { ... }
}
```

**逻辑**：

```python
async def update_agent_config(user_id: str, config: dict):
    # 检查数据库是否有用户记录
    existing = await db.agent_configs.find_one({ "user_id": user_id })

    if existing:
        # 更新记录
        await db.agent_configs.update_one(
            { "user_id": user_id },
            {
                "$set": {
                    "config": config,
                    "is_customized": True,
                    "updated_at": datetime.utcnow()
                }
            }
        )
    else:
        # 创建新记录
        await db.agent_configs.insert_one({
            "user_id": user_id,
            "config": config,
            "is_customized": True,
            "created_at": datetime.utcnow(),
            "updated_at": datetime.utcnow()
        })
```

### 6.3 恢复公共配置

**接口**：`POST /api/trading-agents/agent-config/reset`

**逻辑**：

```python
async def reset_agent_config(user_id: str):
    # 删除数据库中的用户记录
    await db.agent_configs.delete_one({ "user_id": user_id })

    # 后续读取将使用公共配置 YAML
    return { "source": "public" }
```

### 6.4 导出个人配置

**接口**：`GET /api/trading-agents/agent-config/export`

**响应**：YAML 文件下载

**逻辑**：
- 从数据库读取用户配置
- 转换为 YAML 格式
- 返回文件下载

### 6.5 导入个人配置

**接口**：`POST /api/trading-agents/agent-config/import`

**请求**：`multipart/form-data` 上传 YAML 文件

**逻辑**：
- 验证 YAML 格式
- 验证配置结构
- 创建/更新数据库记录

## 七、前端交互

### 7.1 配置状态标识

前端需要展示当前配置来源：

| 状态 | 标识文本 | 说明 |
|------|---------|------|
| 使用个人配置 | 🟢 个人配置 | 数据库有记录，is_customized=true |
| 使用公共配置 | 🔵 公共配置 | 数据库无记录，使用 YAML 文件 |
| 管理员编辑中 | 🟡 编辑模式 | 仅管理员可见，编辑公共配置时显示 |

### 7.2 前端页面差异

**普通用户视角**：
- 配置来源: `[个人配置]` (只读)
- 按钮: `[导出配置]` `[导入配置]` `[恢复公共配置]`
- 提示词编辑: 仅管理员可编辑

**管理员视角**：
- 配置来源: `[个人配置 ▼] / [公共配置]` (可切换)
- 编辑公共配置时:
  - 按钮: `[导出公共配置]` `[导入公共配置]` `[恢复默认配置]`
  - 提示词可编辑
  - 保存直接修改 YAML 文件

### 7.3 配置变更检测

前端需要检测用户是否修改了配置（除了工具权限）：

```
监听配置变化 → 任何字段变更 → 标记为"已修改"
                   └── 显示"保存配置"按钮可用
```

**不触发"已修改"的操作**：
- 展开/折叠智能体详情
- 切换 MCP 服务器复选框（工具权限单独保存）

**触发"已修改"的操作**：
- 修改提示词
- 启用/禁用阶段
- 修改轮数、并发度等参数
- 新增/删除智能体
- 修改智能体名称、描述
