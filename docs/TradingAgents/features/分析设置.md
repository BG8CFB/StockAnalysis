## 一、功能概述

### 1.1 功能说明

分析设置是 TradingAgents 模块中用户自定义分析参数的功能模块，允许用户配置：

- 默认 AI 模型选择
- 辩论轮数配置
- 超时时间设置
- 数据保留策略
- 功能开关

### 1.2 配置对象

**数据库集合**: `trading_agents_settings`

**配置层级**: 用户级别，每个用户独立配置

## 二、配置项说明

### 2.1 默认模型配置

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `data_collection_model_id` | string | - | 数据收集阶段使用的模型 ID（Phase 1） |
| `debate_model_id` | string | - | 辩论阶段使用的模型 ID（Phase 2/3/4） |

**说明**：
- 用户可以在 AI 模型管理中配置多个模型
- 为不同阶段选择不同的模型以优化成本和性能
- 数据收集模型需要调用工具，通常选择较强且较快的模型
- 辩论模型主要是文本生成，通常选择性价比高的模型

### 2.2 辩论配置

| 字段 | 类型 | 范围 | 默认值 | 说明 |
|------|------|------|--------|------|
| `default_debate_rounds` | int | 1-10 | 2 | 默认辩论轮数 |
| `max_debate_rounds` | int | 1-10 | 3 | 最大辩论轮数限制 |

**说明**：
- 辩论轮数影响分析深度和 Token 消耗
- 轮数越多，分析越深入，但成本也越高
- 用户创建任务时可覆盖默认值，但不能超过最大值

### 2.3 超时配置

| 字段 | 类型 | 范围 | 默认值 | 说明 |
|------|------|------|--------|------|
| `phase_timeout_minutes` | int | 5-120 | 30 | 单个阶段超时时间（分钟） |
| `agent_timeout_minutes` | int | 1-60 | 10 | 单个智能体超时时间（分钟） |
| `tool_timeout_seconds` | int | 10-300 | 60 | 工具调用超时时间（秒） |

**说明**：
- 阶段超时：整个阶段的执行时间限制
- 智能体超时：单个智能体的执行时间限制
- 工具超时：单次 MCP 工具调用的超时时间
- 超时后任务标记为失败

### 2.4 数据保留配置

| 字段 | 类型 | 范围 | 默认值 | 说明 |
|------|------|------|--------|------|
| `task_expiry_hours` | int | 1-168 | 24 | 任务过期时间（小时） |
| `archive_days` | int | 1-365 | 30 | 报告归档天数 |

**说明**：
- 任务过期：pending 状态超过该时间自动标记为 expired
- 报告归档：活跃报告超过该天数自动归档到历史表

### 2.5 功能开关

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enable_loop_detection` | bool | true | 启用工具循环检测 |
| `enable_progress_events` | bool | true | 启用 WebSocket 进度事件推送 |
| `enable_agent_thinking` | bool | false | 启用智能体思考过程记录 |

**说明**：
- 循环检测：检测工具调用是否陷入死循环
- 进度事件：通过 WebSocket 推送实时进度
- 思考过程：记录智能体的思考链路（会增加 Token 消耗）

## 三、数据存储

### 3.1 集合结构

**集合名**: `trading_agents_settings`

**文档结构**：

```javascript
{
  "_id": ObjectId("..."),
  "user_id": "user123",
  "settings": {
    // 默认模型
    "data_collection_model_id": "model_id_1",
    "debate_model_id": "model_id_2",

    // 辩论配置
    "default_debate_rounds": 2,
    "max_debate_rounds": 3,

    // 超时配置
    "phase_timeout_minutes": 30,
    "agent_timeout_minutes": 10,
    "tool_timeout_seconds": 60,

    // 数据保留
    "task_expiry_hours": 24,
    "archive_days": 30,

    // 功能开关
    "enable_loop_detection": true,
    "enable_progress_events": true,
    "enable_agent_thinking": false
  },
  "created_at": ISODate("2026-01-14T10:00:00Z"),
  "updated_at": ISODate("2026-01-14T10:00:00Z")
}
```

### 3.2 字段说明

| 字段 | 类型 | 必填 | 索引 | 说明 |
|------|------|------|------|------|
| `_id` | ObjectId | 是 | 主键 | 文档 ID |
| `user_id` | string | 是 | ✓(unique) | 用户 ID |
| `settings` | object | 是 | - | 设置对象 |
| `created_at` | datetime | 是 | - | 创建时间 |
| `updated_at` | datetime | 是 | - | 更新时间 |

## 四、API 接口

### 4.1 读取设置

**接口**: `GET /api/trading-agents/settings`

**响应**：

```javascript
{
  "success": true,
  "data": {
    "data_collection_model_id": "model_id_1",
    "debate_model_id": "model_id_2",
    "default_debate_rounds": 2,
    "max_debate_rounds": 3,
    "phase_timeout_minutes": 30,
    "agent_timeout_minutes": 10,
    "tool_timeout_seconds": 60,
    "task_expiry_hours": 24,
    "archive_days": 30,
    "enable_loop_detection": true,
    "enable_progress_events": true,
    "enable_agent_thinking": false
  }
}
```

### 4.2 更新设置

**接口**: `PUT /api/trading-agents/settings`

**请求体**：完整或部分设置对象

```javascript
{
  "data_collection_model_id": "model_id_2",
  "default_debate_rounds": 3
}
```

**响应**：

```javascript
{
  "success": true,
  "message": "设置已更新"
}
```

### 4.3 恢复默认值

**接口**: `POST /api/trading-agents/settings/reset`

**响应**：

```javascript
{
  "success": true,
  "message": "设置已恢复为默认值",
  "data": { /* 默认设置对象 */ }
}
```

### 5.3 字段验证

| 字段 | 验证规则 | 错误提示 |
|------|----------|----------|
| `default_debate_rounds` | 1-10，≤ max_debate_rounds | 默认轮数必须在 1-10 之间且不超过最大轮数 |
| `max_debate_rounds` | 1-10，≥ default_debate_rounds | 最大轮数必须在 1-10 之间且不小于默认轮数 |
| `phase_timeout_minutes` | 5-120 | 阶段超时必须在 5-120 分钟之间 |
| `agent_timeout_minutes` | 1-60 | 智能体超时必须在 1-60 分钟之间 |
| `tool_timeout_seconds` | 10-300 | 工具超时必须在 10-300 秒之间 |
| `task_expiry_hours` | 1-168 | 任务过期时间必须在 1-168 小时之间 |
| `archive_days` | 1-365 | 归档天数必须在 1-365 天之间 |
