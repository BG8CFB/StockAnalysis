## 一、工具集成概述

### 1.1 工具分类

TradingAgents 智能体使用的工具分为两大类：

| 工具类型 | 数据来源 | 配置方式 | 适用场景 |
|----------|----------|----------|----------|
| **财经工具** | 市场数据模块 | 智能体配置文件 (`enabled_tools`) | 获取股票行情、财务数据等 |
| **MCP 工具** | MCP 服务器 | 智能体配置文件 (`enabled_mcp_servers`) | 扩展功能，自定义数据源 |

### 1.2 集成架构

```

│                           TradingAgents 智能体                           │

│  │                    工具获取与过滤                                │   │
│  │  ┌──────────────────┐        ┌──────────────────┐              │   │
│  │  │  财经工具        │        │  MCP 工具        │              │   │
│  │  │  (本地工具)      │        │  (MCP服务器)     │              │   │
│  │  └──────────────────┘        └──────────────────┘              │   │
```

## 二、财经工具分类

### 2.1 工具列表

财经工具由市场数据模块提供，通过智能体配置文件的 `enabled_tools` 字段配置：

| 工具名称 | 描述 | 数据类型 | 典型用途 |
|----------|------|----------|----------|
| `get_stock_quotes` | 获取股票行情 | OHLCV 数据 | 价格走势分析 |
| `get_technical_indicators` | 获取技术指标 | MACD、RSI、KDJ 等 | 技术分析 |
| `get_financial_statements` | 获取财务报表 | 资产负债表、利润表等 | 基本面分析 |
| `get_financial_indicators` | 获取财务指标 | ROE、ROA、EPS 等 | 财务指标分析 |
| `get_company_info` | 获取公司信息 | 公司资料、业务描述 | 公司研究 |
| `get_company_announcements` | 获取公司公告 | 公告列表、详情 | 消息面分析 |
| `get_macro_economic` | 获取宏观经济数据 | 经济指标 | 宏观环境分析 |

### 2.2 工具配置示例

```yaml
# 技术分析师配置 (phases/phase1/agents/technical.yaml)
enabled_tools:
  - get_stock_quotes           # 获取股票行情
  - get_technical_indicators   # 获取技术指标
```

```yaml
# 基本面分析师配置 (phases/phase1/agents/fundamentals.yaml)
enabled_tools:
  - get_financial_statements   # 获取财务报表
  - get_financial_indicators   # 获取财务指标
  - get_company_info           # 获取公司信息
```

### 2.3 财经工具调用方式

**重要**：财经工具不由 AI 直接调用，而是通过配置后获取信息传递给智能体。

```
用户请求分析股票
      ↓
智能体启动 → 读取 enabled_tools 配置
      ↓
调用市场数据模块获取数据
      ↓
将数据整理后传递给智能体
      ↓
智能体基于数据生成分析报告
```

## 三、MCP 工具集成

### 3.1 MCP 服务器类型

| 服务器类型 | 配置者 | 使用者 | 并发限制 |
|------------|--------|--------|----------|
| **公共服务器** (is_system=True) | 管理员 | 所有用户 | 每用户 10 |
| **个人服务器** (is_system=False) | 用户 | 仅创建者 | 每用户 100 |

### 3.2 MCP 配置方式

在智能体配置文件中，通过 `enabled_mcp_servers` 字段配置 MCP 服务器：

```yaml
# 智能体配置示例
enabled_mcp_servers:
  - name: "FinanceMCP"      # MCP 服务器名称
    required: true          # 是否必需（失败时是否阻止任务）
  - name: "NewsMCP"
    required: false         # 可选服务器，失败时跳过
```

### 3.3 MCP 工具自动发现

MCP 工具通过 MCP 协议自动发现，无需手动配置每个工具：

```
建立 MCP 连接
      ↓
调用 MCP 服务器
      ↓
获取工具列表 (tool_a, tool_b, tool_c, ...)
      ↓
根据智能体配置过滤
      ↓
传递给 LLM
```

### 3.4 连接管理

- **长连接模式**：任务开始时建立，任务完成后 10 秒销毁
- **连接复用**：同一任务内的多个智能体共享连接
- **并发控制**：通过信号量限制每用户的最大并发连接数

## 四、工具配置方式

### 4.1 智能体配置文件

**位置**：`backend/modules/trading_agents/phases/{phase}/agents/{agent}.yaml`

**配置结构**：
```yaml
# 智能体基本信息
slug: technical
name: 技术分析师
enabled: true
model_ref: data_collection_model

# 角色定义（提示词）
role_definition: |
  你是一位专业的技术分析师。
  ...

# 财经工具配置
enabled_tools:
  - get_stock_quotes
  - get_technical_indicators

# MCP 服务器配置
enabled_mcp_servers:
  - name: "FinanceMCP"
    required: true
```

### 4.2 配置优先级

```
用户自定义配置 (is_customized=True)
        ↓
系统公共配置 (user_id="system_public")
        ↓
系统默认配置 (YAML 文件)
```

### 4.3 配置验证

**Phase 1（分析师团队）**：
- ✅ 允许所有修改（添加、删除、编辑工具和服务器）

**Phase 2/3/4（固定智能体）**：
- ✅ 可以修改 `role_definition`（提示词）
- ✅ 可以修改 `enabled`（启用/禁用）
- ❌ 不能修改工具和服务器配置（Phase 2/3/4 不使用工具）

## 五、工具过滤与容错

### 5.1 工具过滤流程

```
MCP 连接建立 → 获取所有工具 [tool_a, tool_b, tool_c, ...]
      ↓
读取智能体配置
      ↓
根据 enabled_mcp_servers 过滤
      ↓
返回过滤后的工具列表
```

### 5.2 容错机制

**必需服务器** (`required: true`)：
- 连接失败 → 抛出异常 → 任务终止
- 适用场景：核心数据源，无法替代

**可选服务器** (`required: false`)：
- 连接失败 → 记录警告 → 任务继续
- 适用场景：辅助数据源，可以降级

### 5.3 工具循环检测

为防止 AI 陷入无限循环调用同一工具，系统实现了工具循环检测：

- **检测阈值**：同一工具连续调用 3 次
- **处理方式**：中断循环，返回错误给智能体
- **实现位置**：`tools/loop_detector.py`

## 六、工具调用流程

### 6.1 完整调用流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│  1. 任务启动                                                             │
│     - 加载用户智能体配置                                                │
│     - 解析 enabled_tools 和 enabled_mcp_servers                         │
└─────────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  2. 获取财经工具                                                         │
│     - 根据配置从市场数据模块获取数据                                    │
│     - 整理后传递给智能体                                                │
└─────────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  3. 获取 MCP 工具                                                        │
│     - 检查 MCP 连接缓存                                                 │
│     - 缓存未命中 → 调用 MCPConnectionPool.acquire_connection()          │
│     - 创建/复用长连接                                                   │
│     - 获取工具列表                                                       │
│     - 根据配置过滤工具                                                  │
└─────────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  4. 创建智能体                                                           │
│     - 合并财经数据和 MCP 工具                                           │
│     - 使用 LangChain create_react_agent()                               │
│     - 绑定 LLM 和工具                                                   │
└─────────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  5. 任务执行（长连接保持）                                               │
│     - LLM 决策调用工具                                                  │
│     - 工具返回结果                                                      │
│     - （重复多次，保持同一连接）                                         │
└─────────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  6. 任务完成                                                             │
│     - connection.mark_complete()                                        │
│     - 启动 10 秒倒计时                                                  │
└─────────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  7. 连接销毁                                                             │
│     - 10 秒后 → connection.close()                                      │
│     - 释放并发槽位                                                      │
└─────────────────────────────────────────────────────────────────────────┘
```
