# 金融数据工具文档

## 概述

本文档描述 TradingAgents 模块可用的金融数据工具。这些工具以 **上下文预加载服务** 的形式实现，**不需要 AI 调用**，也不通过 MCP 协议或 LangChain 工具机制。

---

## 设计模式：上下文预加载

### 核心逻辑

```
┌─────────────────────────────────────────────────────────────┐
│  前端配置页面                                                │
│  - 用户配置智能体需要获取什么信息                             │
│  - 例如：勾选"新闻信息"、"行情数据"、"财务指标"等             │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  工作流启动 (WorkflowScheduler.run)                         │
│  - 读取用户配置的数据需求                                     │
│  - 在 Phase 1 开始前执行数据预加载                           │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  上下文预加载服务 (ContextPreloadService)                    │
│  - 根据配置调用市场数据模块                                   │
│  - 按双通道策略获取数据（用户配置源 vs 公共数据库）            │
│  - 将获取的数据格式化为上下文文本                             │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  上下文注入                                                  │
│  - 将数据注入到 user_prompt 或 WorkflowState.context_data    │
│  - 智能体直接读取上下文进行分析                               │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  智能体执行                                                  │
│  - 智能体无需调用工具                                         │
│  - 直接从上下文中读取预加载的数据进行分析                      │
└─────────────────────────────────────────────────────────────┘
```

### 与现有代码的区别

| 对比项 | 现有代码 (local_tools_adapter.py) | 用户设计 (本文档) |
|--------|-----------------------------------|-------------------|
| 触发方式 | AI 通过 LangChain tool calling 调用 | 前端配置驱动，系统自动预加载 |
| 调用时机 | AI 分析过程中按需调用 | 工作流开始前一次性获取 |
| 数据传递 | 工具返回值 → AI 继续分析 | 数据直接注入上下文 → AI 直接读取 |
| 实现方式 | LangChain BaseTool + MCP 适配 | 直接调用市场数据模块函数 |

## 数据获取策略

工具遵循项目的**双通道数据流**设计：

```
用户配置了数据源（如 TuShare）
    ↓
┌─────────────────────────────────────┐
│ 通道1：实时获取                      │
│ - 直接从用户配置的 API 获取           │
│ - 不存储，不限流                      │
│ - 同时异步存入数据库（通道2）          │
└─────────────────────────────────────┘

用户未配置数据源
    ↓
┌─────────────────────────────────────┐
│ 降级通道：公共数据库                  │
│ - 从公共数据库查询                    │
│ - 数据库无数据时返回空                │
│ - 不触发实时抓取                      │
└─────────────────────────────────────┘
```

### 与现有代码的共存方式

| 文件/模式 | 触发方式 | 使用场景 | 状态 |
|----------|----------|----------|------|
| `local_tools_adapter.py` | AI 通过 LangChain tool calling 调用 | 用户未配置预加载时，AI 按需调用 | 现有，保留 |
| `context_preload/` | 系统在工作流开始前自动预加载 | 用户配置了预加载时，自动注入上下文 | 新增，本文档 |

**兼容性设计**：
- 两种模式可以并存，不会相互冲突
- 用户可以在前端选择使用哪种模式
- 默认使用上下文预加载模式（更高效、更可控）

### 文件职责说明

| 文件 | 职责 |
|------|------|
| `service.py` | `ContextPreloadService` 主服务类，协调各数据提供者，对外提供统一接口 |
| `providers/base.py` | `BaseProvider` 抽象基类，定义数据提供者接口 |
| `providers/*.py` | 各类数据提供者实现，继承 `BaseProvider`，封装具体的 Repository 调用逻辑 |
| `formatters/*.py` | 格式化工具，将数据转换为 Markdown 表格或列表格式 |
| `config.py` | 配置管理，读取用户的数据源配置和预加载配置 |
| `error_handler.py` | 错误处理，实现降级策略和异常捕获 |
| `utils.py` | 工具函数，日期格式化、数据验证、Token 计数等 |

## 工具实现

### 实现方式

工具作为 Python 函数直接调用市场数据模块：

```python
# 不使用 LangChain 工具机制
# 直接调用市场数据模块的 Repository 函数

from core.market_data.repositories.stock_quotes import StockQuoteRepository
from core.market_data.repositories.market_news import MarketNewsRepository

class ContextPreloadService:
    """上下文预加载服务"""

    def __init__(self, user_id: str):
        self.user_id = user_id
        self.quote_repo = StockQuoteRepository()
        self.news_repo = MarketNewsRepository()
        # ... 其他 repositories

    async def preload_context(
        self,
        stock_code: str,
        data_requirements: List[str]
    ) -> Dict[str, Any]:
        """
        预加载上下文数据

        Args:
            stock_code: 股票代码
            data_requirements: 数据需求列表，如 ["news", "quotes", "financials"]

        Returns:
            预加载的上下文数据
        """
        context = {}

        # 根据需求获取数据
        if "news" in data_requirements:
            context["news"] = await self._get_news_context(stock_code)

        if "quotes" in data_requirements:
            context["quotes"] = await self._get_quotes_context(stock_code)

        if "financials" in data_requirements:
            context["financials"] = await self._get_financials_context(stock_code)

        return context

    async def _get_news_context(self, stock_code: str) -> str:
        """获取新闻上下文"""
        # 直接调用 Repository 函数
        news_list = await self.news_repo.get_by_symbol(stock_code, limit=20)

        # 格式化为上下文文本
        context_text = "## 相关新闻\n\n"
        for news in news_list:
            context_text += f"- {news.datetime}: {news.title}\n"
            context_text += f"  {news.content[:200]}...\n\n"

        return context_text

    async def _get_quotes_context(self, stock_code: str) -> str:
        """获取行情上下文"""
        # 直接调用 Repository 函数
        quotes = await self.quote_repo.get_quotes(
            symbol=stock_code,
            start_date="2024-01-01",
            end_date="2024-12-31",
            limit=30
        )

        # 格式化为上下文文本
        context_text = "## 近期行情\n\n"
        for quote in quotes:
            context_text += f"- {quote['trade_date']}: 收盘 {quote['close']}, 涨跌幅 {quote.get('change_pct', 0):.2f}%\n"

        return context_text
```

### 集成到工作流

在 `WorkflowScheduler.run()` 方法中，在 Phase 1 开始前预加载数据：

```python
# backend/modules/trading_agents/workflow/scheduler.py

async def run(self, ...) -> WorkflowState:
    # ... 创建初始状态

    # ========== 新增：上下文预加载 ==========
    from modules.trading_agents.tools.context_preload import ContextPreloadService

    preload_service = ContextPreloadService(user_id=user_id)

    # 从配置中读取数据需求（前端配置的）
    data_requirements = self._get_data_requirements(agent_config)

    # 预加载数据
    context_data = await preload_service.preload_context(
        stock_code=stock_code,
        data_requirements=data_requirements
    )

    # 将数据注入到状态或 prompt 中
    state.context_data = context_data
    # =========================================

    # Phase 1: 信息收集与基础分析
    # 智能体直接从 state.context_data 读取数据，无需调用工具
    ...
```

#### 1.1 日线行情上下文

**函数名**: `get_quotes_context(stock_code: str, days: int = 30) -> str`

**功能**: 获取股票日线行情并格式化为上下文文本

**参数**:
| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|

**返回格式**:

## 前端配置

### 智能体配置对话框

**配置位置**：智能体配置在**编辑智能体对话框**中进行，路径为：`TradingAgents 设置 → 选择阶段 → 编辑智能体 → 本地工具配置`

在编辑智能体对话框中，新增"**本地工具配置**"部分（与"启用MCP"并列），用户可以：

1. **选择数据类型**：勾选智能体需要预加载到上下文的数据类型
   - ☑ 新闻信息
   - ☑ 行情数据
   - ☑ 财务报表
   - ☑ 财务指标
   - ☑ 公司信息
   - ☑ 技术指标
   - ☑ 宏观经济

2. **配置数据参数**：设置默认查询参数
   - 时间范围（最近 30 天 / 60 天 / 90 天）
   - 数据数量（20 条 / 50 条 / 100 条）

3. **保存配置**：配置保存到智能体配置中，工作流启动时自动读取

### 配置数据结构

智能体的 YAML 配置文件中新增 `local_tools` 字段：

```yaml
# 示例：基本面分析师配置
```

## 数据库表结构

工具使用的公共数据库表（**已实现，可直接调用**）：

| 表名 | Repository | 主要查询函数 | 主要字段 |
|------|-----------|-------------|----------|
| `stock_quotes` | `StockQuoteRepository` | `get_quotes()`, `get_latest_quote()` | symbol, trade_date, open, high, low, close, volume, amount |
| `stock_info` | `StockInfoRepository` | `get_by_symbol()`, `get_by_market()` | symbol, name, industry, sector, listing_date, exchange |
| `stock_financials` | `StockFinancialRepository` | `get_financial_history()`, `get_income_statement()` | symbol, report_date, report_type, revenue, net_profit, total_assets |
| `stock_financial_indicators` | `StockFinancialIndicatorRepository` | `get_indicator_history()`, `get_latest_indicator()` | symbol, end_date, roe, roa, debt_to_assets, net_profit_margin |
| `market_news` | `MarketNewsRepository` | `get_by_symbol()`, `get_latest_news()` | news_id, symbol, title, content, datetime, source |
| `macro_economic` | `MacroEconomicRepository` | `get_macro_history()`, `get_cpi()`, `get_gdp()` | indicator, period, value |

**说明**：
- 所有 Repository 都已在 `backend/core/market_data/repositories/` 目录下实现
- 工具函数可以直接调用这些 Repository 方法获取数据
- 无需重新实现数据库访问逻辑

### 1. 错误处理和降级策略

#### 错误处理层次

```
┌─────────────────────────────────────────────────────────────┐
│  第1层：Provider 层错误处理                                  │
│  - 捕获 Repository 调用异常                                  │
│  - 记录 ERROR 日志（包含 user_id, stock_code, 异常信息）       │
│  - 返回空数据而非抛出异常                                     │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  第2层：Service 层降级策略                                   │
│  - 用户数据源失败 → 降级到公共数据库                          │
│  - 公共数据库失败 → 返回空上下文（附带说明）                   │
│  - 记录降级原因到 state.context_data_metadata                 │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  第3层：工作流层容错                                         │
│  - 空上下文不影响工作流继续                                   │
│  - 在智能体 Prompt 中说明数据缺失情况                         │
└─────────────────────────────────────────────────────────────┘
```

```python
# context_preload/error_handler.py

class ErrorHandler:
    """错误处理器"""

    async def handle_provider_error(
        self,
        error: Exception,
        provider_name: str,
        stock_code: str,
        user_id: str
    ) -> Dict[str, Any]:
        """
        处理 Provider 层错误

        Returns:
            包含错误信息和空数据的字典
        """
        logger.error(
            f"[ContextPreload] {provider_name} failed for {stock_code}, "
            f"user={user_id}, error={error}"
        )

        return {
            "data": None,
            "error": str(error),
            "provider": provider_name,
            "fallback_used": False
        }

    async def try_with_fallback(
        self,
        primary_func: Callable,
        fallback_func: Callable,
        context: Dict[str, Any]
    ) -> Dict[str, Any]:
        """
        尝试主函数，失败时降级到备用函数

        Args:
            primary_func: 主函数（用户数据源）
            fallback_func: 备用函数（公共数据库）
            context: 上下文信息

        Returns:
            结果字典，包含数据和降级信息
        """
        try:
            # 尝试主函数
            result = await primary_func()
            return {
                "data": result,
                "source": "primary",
                "fallback_used": False
            }
        except Exception as e:
            logger.warning(f"Primary function failed: {e}, trying fallback")

            # 降级到备用函数
            try:
                result = await fallback_func()
                return {
                    "data": result,
                    "source": "fallback",
                    "fallback_used": True,
                    "error": str(e)
                }
            except Exception as fallback_error:
                logger.error(f"Fallback function also failed: {fallback_error}")
                return {
                    "data": None,
                    "source": "none",
                    "fallback_used": False,
                    "error": f"Primary: {e}, Fallback: {fallback_error}"
                }
```

```python
# context_preload/config.py

class ContextPreloadConfig:
    """上下文预加载配置管理"""

    async def get_config(
        self,
        user_id: str,
        agent_slug: Optional[str] = None
    ) -> Dict[str, Any]:
        """
        获取用户的上下文预加载配置

        Args:
            user_id: 用户 ID
            agent_slug: 智能体 slug（可选，用于获取覆盖配置）

        Returns:
            配置字典
        """
        # 1. 获取用户默认配置
        default_config = await self._get_user_default_config(user_id)

        # 2. 如果指定了智能体，应用智能体级别覆盖
        if agent_slug:
            agent_overrides = await self._get_agent_overrides(user_id, agent_slug)
            if agent_overrides:
                default_config = self._merge_config(default_config, agent_overrides)

        return default_config

    def _merge_config(
        self,
        base_config: Dict[str, Any],
        overrides: Dict[str, Any]
    ) -> Dict[str, Any]:
        """合并配置（深合并）"""
        result = base_config.copy()
        for key, value in overrides.items():
            if key in result and isinstance(result[key], dict) and isinstance(value, dict):
                result[key] = self._merge_config(result[key], value)
            else:
                result[key] = value
        return result
```

### 3. 上下文长度管理

#### Token 计数和长度限制

```python
# context_preload/utils.py

class ContextLengthManager:
    """上下文长度管理器"""

    # 各类数据的 Token 估算系数（中文）
    TOKEN_RATIOS = {
        "quotes": 0.3,      # 表格格式，系数较低
        "news": 1.0,        # 纯文本，系数较高
        "financials": 0.4,  # 表格格式
        "company": 0.2,     # 简短信息
        "macro": 0.8        # 文本为主
    }

    # 最大 Token 限制（预留空间给智能体输出）
    MAX_CONTEXT_TOKENS = 12000  # 假设模型上限 16K，预留 4K

    def estimate_tokens(self, text: str, data_type: str) -> int:
        """
        估算文本的 Token 数量

        Args:
            text: 文本内容
            data_type: 数据类型

        Returns:
            估算的 Token 数量
        """
        # 简单估算：中文字符数 × 系数
        char_count = len(text)
        ratio = self.TOKEN_RATIOS.get(data_type, 1.0)
        return int(char_count * ratio)

    def check_and_truncate(
        self,
        context: Dict[str, str],
        max_tokens: Optional[int] = None
    ) -> Dict[str, Any]:
        """
        检查上下文长度，必要时截断

        Args:
            context: 上下文数据字典
            max_tokens: 最大 Token 限制

        Returns:
            处理后的上下文和元数据
        """
        max_tokens = max_tokens or self.MAX_CONTEXT_TOKENS

        # 计算总 Token
        total_tokens = 0
        for key, value in context.items():
            total_tokens += self.estimate_tokens(value, key)

        # 如果未超限，直接返回
        if total_tokens <= max_tokens:
            return {
                "context": context,
                "truncated": False,
                "total_tokens": total_tokens
            }

        # 超限时，按优先级截断
        priority = ["company", "quotes", "financials", "indicators", "news", "macro"]
        truncated_context = {}
        current_tokens = 0

        for key in priority:
            if key not in context:
                continue

            value = context[key]
            tokens = self.estimate_tokens(value, key)

            if current_tokens + tokens <= max_tokens:
                truncated_context[key] = value
                current_tokens += tokens
            else:
                # 截断当前项
                remaining_tokens = max_tokens - current_tokens
                truncated_text = self._truncate_text(value, remaining_tokens, key)
                truncated_context[key] = truncated_text
                break

        return {
            "context": truncated_context,
            "truncated": True,
            "total_tokens": current_tokens,
            "original_tokens": total_tokens
        }

    def _truncate_text(self, text: str, max_tokens: int, data_type: str) -> str:
        """截断文本到指定 Token 数量"""
        ratio = self.TOKEN_RATIOS.get(data_type, 1.0)
        max_chars = int(max_tokens / ratio)

        if len(text) <= max_chars:
            return text

        # 截断并添加提示
        truncated = text[:max_chars]
        return truncated + "\n\n...(数据已截断，完整数据请在数据库中查看)"
```

### 4. 技术指标计算方案

技术指标不需要从外部 API 获取，而是基于行情数据**本地计算**：

```python
# context_preload/providers/technical.py

class TechnicalProvider(BaseProvider):
    """技术指标提供者（本地计算）"""

    def __init__(self, user_id: str, quotes_provider: "QuotesProvider"):
        super().__init__(user_id)
        self.quotes_provider = quotes_provider  # 依赖行情提供者

    async def get_technical_indicators(
        self,
        stock_code: str,
        indicators: List[str],
        days: int = 30
    ) -> Dict[str, Any]:
        """
        计算技术指标

        Args:
            stock_code: 股票代码
            indicators: 指标列表，如 ["MA5", "MA10", "MACD", "KDJ", "RSI"]
            days: 计算周期

        Returns:
            技术指标数据
        """
        # 1. 获取基础行情数据
        quotes = await self.quotes_provider.get_quotes(stock_code, days=days)

        if not quotes:
            return {}

        # 2. 转换为 DataFrame（用于计算）
        import pandas as pd
        df = pd.DataFrame(quotes)

        # 3. 计算各类指标
        result = {}

        if "MA" in indicators or any(i.startswith("MA") for i in indicators):
            result["MA"] = self._calculate_ma(df, [5, 10, 20, 60])

        if "MACD" in indicators:
            result["MACD"] = self._calculate_macd(df)

        if "KDJ" in indicators:
            result["KDJ"] = self._calculate_kdj(df)

        if "RSI" in indicators:
            result["RSI"] = self._calculate_rsi(df)

        if "BOLL" in indicators:
            result["BOLL"] = self._calculate_boll(df)

        return result

    def _calculate_ma(self, df: pd.DataFrame, periods: List[int]) -> Dict[str, List[float]]:
        """计算移动平均线"""
        result = {}
        close = df["close"]
        for period in periods:
            result[f"MA{period}"] = close.rolling(window=period).mean().tolist()
        return result

    def _calculate_macd(self, df: pd.DataFrame) -> Dict[str, List[float]]:
        """计算 MACD"""
        close = df["close"]
        ema12 = close.ewm(span=12).mean()
        ema26 = close.ewm(span=26).mean()
        dif = ema12 - ema26
        dea = dif.ewm(span=9).mean()
        macd = (dif - dea) * 2

        return {
            "DIF": dif.tolist(),
            "DEA": dea.tolist(),
            "MACD": macd.tolist()
        }

    def _calculate_kdj(self, df: pd.DataFrame, n: int = 9) -> Dict[str, List[float]]:
        """计算 KDJ"""
        high = df["high"]
        low = df["low"]
        close = df["close"]

        low_min = low.rolling(window=n).min()
        high_max = high.rolling(window=n).max()

        rsv = (close - low_min) / (high_max - low_min) * 100
        k = rsv.ewm(com=2).mean()
        d = k.ewm(com=2).mean()
        j = 3 * k - 2 * d

        return {
            "K": k.tolist(),
            "D": d.tolist(),
            "J": j.tolist()
        }

    def _calculate_rsi(self, df: pd.DataFrame, period: int = 12) -> List[float]:
        """计算 RSI"""
        close = df["close"]
        delta = close.diff()

        gain = (delta.where(delta > 0, 0)).rolling(window=period).mean()
        loss = (-delta.where(delta < 0, 0)).rolling(window=period).mean()

        rs = gain / loss
        rsi = 100 - (100 / (1 + rs))

        return rsi.tolist()

    def _calculate_boll(self, df: pd.DataFrame, n: int = 20) -> Dict[str, List[float]]:
        """计算布林带"""
        close = df["close"]
        mid = close.rolling(window=n).mean()
        std = close.rolling(window=n).std()
        upper = mid + 2 * std
        lower = mid - 2 * std

        return {
            "UPPER": upper.tolist(),
            "MID": mid.tolist(),
            "LOWER": lower.tolist()
        }
```
